# 技术术语与设计决策

术语不按字典背诵，而按“定义 → 风险 → 工程动作 → 案例”使用。

| 术语 | 核心含义 | 工程动作 |
|---|---|---|
| 防腐层 ACL | 隔离外部/旧系统模型 | 在 Adapter 中转换，不污染领域对象 |
| DTO/VO/PO/Entity | 协议、值、持久化和身份对象 | 分离边界，避免表结构直通 API |
| Aggregate | 维护一组业务不变量的边界 | 控制事务、锁和并发范围 |
| Context | DDD 的业务边界或 Agent 的决策信息 | 明确范围、生命周期和裁剪策略 |
| 幂等 | 重复操作结果可预测 | 业务号、唯一约束、状态判断、消费记录 |
| 鲁棒性 | 面对重复、乱序、缺字段、超时仍可控 | 超时、重试上限、降级、补偿和审计 |
| 大 Key | 单个 Redis 数据过大 | 拆分、限制返回量、关注网络和迁移 |
| 热 Key | 访问集中在少数 key | 多级缓存、分片、限流、请求合并 |
| TP99 | 99 分位尾延迟 | 与吞吐、错误率、资源和依赖耗时一起分析 |
| 埋点 | 记录业务动作、上下文和结果 | 先定义事件字典，再采集、看板和复盘 |

## 使用方式

每次设计评审至少回答：这个概念解决什么问题、引入什么代价、失败时怎么办、如何验证、哪个案例证明它有效。
