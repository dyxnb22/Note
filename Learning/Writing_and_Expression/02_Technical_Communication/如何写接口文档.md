# 如何写接口文档

## 主题定位

这篇讲的是 `如何写接口文档`。它的目标不是教你一招临场小技巧，而是把一个高频表达问题沉淀成可复用的方法，让你在写作、汇报、沟通和面试里都能稳定用出来。

## 核心概念

- 接口文档首先回答怎么接、会出什么错、边界在哪里。
- 字段、示例、错误码和兼容性说明经常比概述更重要。
- 好接口文档能显著降低协作摩擦。

## 核心机制

- 接入方需要先有可执行信息。
- 边界不清会引发大量额外沟通。
- 示例可以快速降低误解概率。

## 实际例子

- HTTP API。
- SDK 接入说明。
- 内部服务接口。
## 最小练习

- 选一次最近的表达，判断它最大的问题是在对象、结构还是信息取舍。
- 用 3 句话重新解释一次 `如何写接口文档`：一句讲问题，一句讲方法，一句讲价值。
- 为同一段内容做两个版本：写给熟悉背景的人，和写给不熟悉背景的人。

## 一页总结

`如何写接口文档` 的核心，不在于让语言更花，而在于让表达更有方向感、对象感和可执行性。只要你能稳定做到“对象更清楚、结构更明确、重点更前置”，很多表达问题都会一起改善。
