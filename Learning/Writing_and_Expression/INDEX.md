# 写作与表达索引

这条主线训练的不是辞藻，而是把理解组织成别人能判断、复用并采取行动的信息。

## 学习路径

```text
表达目标与读者
  → 结构化写作
  → 技术和工作场景
  → 论证、异议与行动
```

## 场景速查

- [表达基础速查](00_Foundations/表达基础速查.md)：读者、抽象层级、复杂度和信息密度。
- [结构化写作速查](01_Structured_Writing/结构化写作速查.md)：提纲、例子、节奏、压缩和复盘。
- [技术沟通速查](02_Technical_Communication/技术沟通速查.md)：接口、流程、图表、事故与决策记录。
- [职场沟通速查](03_Workplace_Communication/职场沟通速查.md)：会议、上下沟通、跨团队和冲突。
- [论证与说服速查](04_Persuasion_and_Argument/论证与说服速查.md)：异议、不确定性、故事、让步和关系。

## 使用方式

遇到具体任务时先选择对应主文档，再用速查补漏：

- 写方案：设计文档 → 技术沟通速查。
- 做汇报：工作汇报 → 职场沟通速查。
- 写总结或复盘：总结框架 → 结构化写作速查。
- 面对质疑：有效论证 → 论证与说服速查。

每次练习至少保留“原稿、修改稿、修改理由、真实反馈”四项。没有前后对照的泛泛阅读，难以形成稳定能力。
