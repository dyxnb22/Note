# 职场经验与成长

这里把 39 条职场材料合并成可执行的判断框架。

## 工作与成长

- 先做好当前交付，再用复盘、文档、源码和真实问题建立长期能力。
- 工作三到五年后，从“看课程”转向官方资料、源码、Issue、PR、案例和自己的问题清单。
- 深耕业务和深耕技术都可以形成壁垒，关键是能把业务价值、技术判断和交付结果连接起来。
- 架构能力不等于画图或做 PPT，还包括复杂问题解决、跨团队推进、评审、培养、组件和结果负责。

## 选择与迁移

- 判断转岗、跳槽、技术/业务路线时，看下一站能获得什么能力、资源和证据。
- 工作内容、团队反馈、导师质量、业务健康、学习密度、收入和生活约束一起考虑。
- 迁移不是逃避当前问题；先写清要解决的问题、预计停留周期和复盘标准。

## 提升方法

1. 每周补齐一个任务的上下游、数据状态、依赖和异常边界。
2. 每个项目形成架构图、问题清单、结果指标和一段 STAR 表达。
3. 学习采用“资料/源码 → 小案例 → 项目应用 → 复盘分享”。
4. 晋升和求职材料只写真实贡献，把业务价值、技术深度、协作影响和结果证据放在同一条链上。

## 分阶段重点

- 在校：计算机基础、语言、数据库、Web、Git 和小项目。
- 1–3 年：补业务上下文、MQ、RPC、缓存、服务治理、部署和排障。
- 3–5 年：深入 JVM/JUC、Spring、Netty、字节码、监控、容器和复杂场景设计。
- 更高阶段：承担架构决策、跨团队协调、标准建设、技术迁移和人才培养，同时保持一个领域的技术深度。
