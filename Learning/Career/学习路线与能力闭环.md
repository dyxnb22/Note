# 学习路线与能力闭环

主线见[Learning 总索引](../INDEX.md)。学习不以看完目录为完成标准，而以“能解释、能设计、能验证、能迁移、能表达”为闭环。

## 分阶段目标

| 阶段 | 最小产出 |
|---|---|
| 基础 | 能读代码、定位问题、解释数据和网络边界 |
| 后端 | 能完成一个有状态、可测试、可部署的业务链路 |
| 系统设计 | 能说明容量、边界、一致性、失败和取舍 |
| AI 工程 | 能构建工具调用/检索/工作流，并控制权限和成本 |
| 项目验证 | 能从 README、源码、配置和测试给出证据等级 |
| 求职输出 | 能用业务目标、难点、方案、结果和边界讲项目 |

## 学习动作

每周选择一个真实问题，完成：资料/源码 → 最小案例 → 项目应用 → 失败复盘 → 可追问表达。笔记只保留判断框架、关键例子和边界，不复制课程目录。
