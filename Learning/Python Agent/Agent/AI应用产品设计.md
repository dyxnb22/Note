# AI 应用产品设计

## 先判断是否需要 AI

适合 AI 的是输入开放、结果允许概率性、人工规则成本高且可通过反馈改善的问题。若规则明确、错误代价极高或需要严格确定性，优先传统流程；可把 AI 放在提取、排序、草稿或解释环节，而非直接授予最终执行权。

## 定义任务与成功

写清用户是谁、触发场景、输入/输出、失败后果、人工介入点和衡量指标。指标要同时含任务质量（正确、覆盖、可用）、效率（时延、完成率）和风险（误操作、投诉、越权）。“模型回答得像人”不是产品指标。

## 交互与信任

告诉用户系统在做什么、依据何在、何时需要确认；长任务应展示状态和可取消点。高影响操作必须预览、确认、可撤销并留审计。不要用拟人化措辞掩盖模型不确定性。

## 迭代闭环

将用户反馈、失败样本、trace 与版本关联；先修复高频且高损失的失败类别。离线 eval 保障回归，线上灰度验证真实价值。产品需求应反向约束 RAG、工具、权限和模型选择。
