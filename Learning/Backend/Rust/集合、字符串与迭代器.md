# 集合、字符串与迭代器

## 1. `Vec<T>`

`Vec<T>` 是堆上连续、可增长的同类型元素集合。它拥有元素，扩容时可能重新分配内存。

```rust
let mut values = Vec::new();
values.push(10);
values.push(20);

let first: Option<&i32> = values.get(0);
let last = values.pop();
```

如果大致知道容量，可以使用 `Vec::with_capacity` 减少扩容：

```rust
let mut values = Vec::with_capacity(100);
```

遍历时要明确是否移动、借用还是可变借用：

```rust
for value in &values { /* 只读借用 */ }
for value in &mut values { *value += 1; }
for value in values { /* 消费 Vec */ }
```

## 2. 字符串

- `String`：拥有、可增长的 UTF-8 字节缓冲区。
- `&str`：字符串切片，通常是对已有 UTF-8 数据的借用。
- `char`：一个 Unicode 标量值，不代表一个用户感知的完整字符。
- `u8`：一个字节，不一定是完整字符。

```rust
let owned = String::from("hello");
let view: &str = &owned;
let combined = format!("{view} Rust");
```

不能直接使用 `s[0]`，因为一个 Unicode 字符可能占多个字节。根据需求选择：

```rust
let first_byte = text.as_bytes().first();
let first_char = text.chars().next();
let graphemes = text.chars().count();
```

`chars()` 按 Unicode 标量值遍历，不一定按用户看到的字素簇遍历；真正的文本编辑可能需要专门的 Unicode crate。

常用字符串操作：`push`、`push_str`、`split`、`lines`、`trim`、`contains`、`starts_with`、`replace`、`parse`。

## 3. 切片

切片是对连续序列的借用视图：

```rust
fn sum(values: &[i32]) -> i32 {
    values.iter().sum()
}
```

`&[T]` 比 `&Vec<T>` 更通用，数组和 Vec 都可以传入。字符串切片的边界必须位于 UTF-8 字符边界，否则切片会 panic：

```rust
let prefix = &text[..text.len().min(5)]; // 不保证 5 是字符边界
```

处理外部文本时不要用字节下标猜字符边界。

## 4. `HashMap` 与 `HashSet`

```rust
use std::collections::{HashMap, HashSet};

let mut counts = HashMap::new();
for word in words {
    *counts.entry(word).or_insert(0) += 1;
}

let mut seen = HashSet::new();
seen.insert("rust");
```

`HashMap::entry` 适合“存在则修改，不存在则插入”。键必须实现 `Eq` 和 `Hash`，放入 map 后不能以会影响哈希的方式改变键。

需要稳定排序或有序遍历时考虑 `BTreeMap`/`BTreeSet`；不要假设 `HashMap` 的遍历顺序。

## 5. 迭代器

迭代器是惰性的；调用 `map`、`filter` 不会立刻执行，直到 `collect`、`sum`、`for_each` 等消费操作发生：

```rust
let result: Vec<_> = values
    .iter()
    .filter(|value| **value > 0)
    .map(|value| value * 2)
    .collect();
```

常用适配器：

- `map`：转换元素。
- `filter`：保留满足条件的元素。
- `filter_map`：同时过滤和转换 `Option`。
- `flat_map`：展开嵌套迭代器。
- `enumerate`：附加索引。
- `zip`：配对两个迭代器。
- `take`、`skip`、`chain`、`peekable`：控制遍历。

常用消费者：`collect`、`sum`、`count`、`any`、`all`、`find`、`position`、`fold`。

## 6. `iter`、`iter_mut` 与 `into_iter`

- `iter()` 产生 `&T`，通常不消费集合。
- `iter_mut()` 产生 `&mut T`，允许修改元素。
- `into_iter()` 取得集合所有权并产生 `T`。

在不同 Rust edition 和集合类型上，`array.into_iter()` 的历史行为容易混淆；需要时显式写出借用或消费意图。

## 7. `collect` 的类型推断

```rust
let numbers: Result<Vec<i32>, _> = text
    .split(',')
    .map(str::parse)
    .collect();
```

这里 `collect` 可以把多个 `Result` 聚合成一个 `Result<Vec<_>, _>`，遇到第一个错误就停止。复杂链条应给中间结果写类型，帮助编译器和读者理解。

## 8. 性能与可读性

迭代器通常可以被优化为与手写循环相近的代码，但不要为了“函数式”强行写出难读的长链。选择依据：

- 逻辑简单、转换清晰：迭代器。
- 需要多个可变状态、提前退出或复杂调试：循环。
- 不确定性能：先写清晰版本，再基准测试。

避免在循环中反复分配字符串；可预分配容量、复用缓冲区，或使用 `String::with_capacity`。
