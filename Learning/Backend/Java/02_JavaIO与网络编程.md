# Java I/O 与网络编程

本页负责 Java 中的数据传输、序列化和 I/O 模型。TCP、HTTP 等协议原理见 [计算机网络](../../CS/03_Network.md)，Netty 等框架应建立在本页的 Stream、Channel、Buffer、Selector 和线程模型之上。

## 对象如何跨 JVM 传输

- Java 序列化：使用 `ObjectOutputStream` 和 `ObjectInputStream` 传输字节流。
- 消息传递：通过 MQ 或网络协议传输序列化后的数据。
- RPC：使用 gRPC 等框架调用另一个 JVM 的服务。
- 共享存储：将数据写入数据库或 Redis，由另一个 JVM 读取。

跨服务传输时通常使用明确、可演进的跨语言协议，而不是直接传输 Java 内存对象。

## 为什么通常不推荐 Java 原生序列化

Java 原生序列化的问题包括：

- 跨语言能力差。
- 数据体积和性能通常不如专用协议。
- 类演进和 `serialVersionUID` 容易造成兼容问题。
- 反序列化存在安全风险，不能对不可信输入直接调用 `readObject()`。

实际项目通常根据场景使用 JSON、Protobuf 或 Avro。Protobuf 需要 `.proto` 定义，体积小、编解码性能好，适合服务间通信；JSON 可读性好，但同样需要 Schema、字段兼容和输入限制。

## BIO、NIO、AIO

| 模型 | 特点 | 核心概念 |
| --- | --- | --- |
| BIO | 同步阻塞，代码简单，但大量连接通常需要大量线程 | Stream、Socket |
| NIO | 同步非阻塞，可用多路复用监听多个连接 | Channel、Buffer、Selector |
| AIO | 异步 I/O，操作完成后通知应用 | CompletionHandler、Future |

NIO 的“非阻塞”指读写调用不会一直等待；“同步”指应用线程仍需主动获取就绪事件并处理读写。AIO 则由异步操作完成后通知应用。

I/O 模型与线程模型不是同一个维度。BIO 也可以使用线程池，NIO 也需要线程执行编解码和业务逻辑；关键是等待 I/O 时线程是否被占用。

## Stream

字节流处理二进制数据，字符流在字节与字符之间进行编码转换。使用字符流时必须显式确定字符集，不能依赖机器默认编码。

装饰器组合是 Java I/O 的核心结构：底层流负责数据源，缓冲流提高效率，数据流或对象流提供更高层编码。所有资源优先使用 `try-with-resources` 关闭。

## Channel、Buffer 与 Selector

- Channel 表示可读写通道，可以双向操作。
- Buffer 保存待读取或写出的数据，`position`、`limit` 和 `capacity` 决定当前可操作区间。
- Selector 让一个线程监听多个 Channel 的就绪事件。

非阻塞读写必须正确处理：

- 一次读不到完整消息。
- 一次写不完全部 Buffer。
- 返回 0 不等于连接关闭。
- 返回 -1 表示对端正常关闭输入流。
- 消息边界需要长度字段、分隔符或固定长度协议。

## 文件 I/O

优先使用 `java.nio.file.Path` 和 `Files`。大文件不要一次性读入内存；按流或 Channel 分块处理，并限制上传大小、路径和并发。

写重要文件时可采用“写临时文件 → flush/fsync → 原子替换”的模式。跨文件系统移动不一定原子，错误处理和恢复策略必须明确。

## 网络服务的线程模型

典型 Reactor 模型把接收连接、I/O 事件和业务执行分开。事件循环线程不应执行阻塞数据库查询、远程调用或长时间 CPU 任务，否则会拖慢同一循环上的其他连接。

需要明确：

- Acceptor 和 I/O Worker 数量。
- 编解码和业务逻辑在哪个线程池执行。
- 写队列上限和背压。
- 空闲连接、读写和请求级超时。
- 半关闭、断线和重连策略。

## 虚拟线程

Java 21 虚拟线程降低大量阻塞式 I/O 任务的线程创建和调度成本，使“一个请求一个线程”的代码更容易扩展。它不提升 CPU 密集计算，也不会消除数据库连接、下游并发和内存等资源上限。

使用虚拟线程仍要限制外部并发，并检查 `synchronized`、本地调用等可能导致载体线程固定的问题。

## 验收清单

- 能区分阻塞/非阻塞与同步/异步。
- 能解释 NIO 为什么仍然需要线程执行实际读写和业务逻辑。
- 能正确处理半包、粘包、短写和连接关闭。
- 能为序列化协议定义兼容、安全和大小限制。
- 能解释虚拟线程改善什么、不能改善什么。
