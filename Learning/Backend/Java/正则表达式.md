# 正则表达式

正则表达式用于按规则匹配、提取或替换字符串。面试和业务中重点掌握边界、分组、量词和 Java API，不必死记复杂表达式。

## 常用语法

| 写法 | 含义 |
| --- | --- |
| `.` | 任意单个字符（默认不含换行） |
| `\d`、`\w`、`\s` | 数字、单词字符、空白字符 |
| `[abc]`、`[^abc]` | 字符集合、非集合字符 |
| `^`、`$` | 字符串/行的开始、结束 |
| `*`、`+`、`?` | 重复 0 次以上、1 次以上、0 或 1 次 |
| `{m,n}` | 重复 m 到 n 次 |
| `(...)`、`(?:...)` | 捕获分组、非捕获分组 |
| `|` | 或 |

量词默认是贪婪的，`*?`、`+?`、`{m,n}?` 可改为尽量少匹配。`(?=...)`/`(?!...)` 是正向/负向前瞻，用来做条件约束但不消耗字符；反向引用如 `\1` 可匹配前面捕获的同样内容。

## Java 中怎么使用？

```java
Pattern pattern = Pattern.compile("\\d+");
Matcher matcher = pattern.matcher("order-123");
if (matcher.find()) {
    String value = matcher.group();
}
```

- `matches()` 要求整个输入匹配；`find()` 查找任意子串；`lookingAt()` 从输入开头尝试匹配。
- 循环 `find()` 读取多个匹配，捕获组通过 `group(1)` 获取。
- 固定表达式可复用 `Pattern`，避免每次调用都重新编译；用户输入拼接前使用 `Pattern.quote()`，避免被当成正则语法。

## 正则有什么性能和安全风险？

复杂嵌套量词和大量回溯可能导致 ReDoS。对外部输入应限制长度，避免 `(a+)+` 一类高回溯模式，必要时拆成更明确的解析逻辑并设置超时/隔离。正则适合词法校验和提取，不适合解析嵌套结构或替代完整 JSON/XML 解析器。
