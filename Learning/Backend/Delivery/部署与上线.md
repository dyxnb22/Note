# 部署与上线
部署的目标不是“能跑一次”，而是让服务在服务器上稳定、可配置、可重启、可观察、可更新。

## 1. 部署前先明确

上线前先回答：

- 这个服务监听哪个端口？
- 配置从哪里来？
- 数据存在哪里？
- 日志怎么看？
- 崩了怎么重启？
- 如何健康检查？
- 如何更新版本？
- API Key 是否安全？

## 2. Linux 基础

常用命令：

```bash
pwd                         # 当前目录
ls -la                      # 查看文件，包括隐藏文件
cd /opt/app                 # 进入项目目录
cat .env                    # 查看环境变量文件，注意不要随便发给别人
ps aux | grep uvicorn       # 查看 uvicorn 进程
ss -lntp                    # 查看端口监听情况
tail -f app.log             # 实时查看日志
```

权限：

```bash
chmod +x scripts/start.sh       # 给脚本执行权限
chown -R appuser:appuser /opt/app # 修改目录所属用户
```

不要长期用 root 直接跑业务服务。

## 3. 环境变量

`.env` 放真实配置，`.env.example` 放模板。

```plain text
OPENAI_API_KEY=...
MODEL_NAME=gpt-5
DATABASE_URL=postgresql://user:pass@db:5432/app
REDIS_URL=redis://redis:6379/0
```

原则：

- `.env` 不提交 Git。
- 线上和本地用不同配置。
- secrets 不写进 Dockerfile。
- 日志不打印 secrets。

## 4. Uvicorn / Gunicorn

开发：

```bash
uvicorn app.main:app --reload
```

生产基础版：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```
