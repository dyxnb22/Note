# Linux 命令详解指南

## Linux 命令面试速查

面试回答 Linux 命令题时，不要只背命令名，要能说清楚：命令解决什么问题、常用参数是什么、实际排查时如何组合使用。

------

### 1. 文件与目录操作

| 命令    | 核心参数                                                     | 典型用法                                                     |
| ------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `ls`    | `-l` 长格式，`-a` 显示隐藏文件，`-h` 人性化大小，`-t` 按修改时间排序 | `ls -lah` 查看权限、所有者、大小和隐藏文件；`ls -lht` 按时间倒序查看最近修改文件。 |
| `cd`    | `-` 回到上一个目录，`~` 回到家目录                           | `cd -` 在两个目录间快速切换。                                |
| `mkdir` | `-p` 递归创建父目录                                          | `mkdir -p project/src/main`，父目录不存在时自动创建。        |
| `cp`    | `-r` 递归复制，`-p` 保留权限和时间戳                         | `cp -rp source_dir/ backup_dir/` 复制目录并保留原属性。      |
| `rm`    | `-r` 递归，`-f` 强制，`-i` 删除前确认                        | `rm -i file.txt` 删除前确认；`rm -rf dir/` 强制删除目录，生产环境慎用。 |
| `ln`    | `-s` 创建软链接                                              | `ln -s /usr/local/bin/python3 /usr/bin/python` 创建快捷入口。 |

**面试重点：软链接和硬链接**

-   软链接类似快捷方式，保存的是目标路径，可以跨文件系统；目标文件删除后软链接失效。
-   硬链接指向同一个 inode，不能跨文件系统，通常不能直接给目录创建硬链接；删除原文件后，只要还有硬链接，数据仍然存在。

### 2. 文本查看与处理

Linux 中配置、日志、进程信息大多可以按文本处理，常见排查通常围绕 `cat`、`less`、`tail`、`grep`、`awk`、`sed` 组合展开。

| 命令   | 核心参数                                                 | 典型用法                                                     |                                                              |
| ------ | -------------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `cat`  | 直接输出文件内容                                         | `cat config.yml` 查看短文件。长文件不建议直接 `cat`。        |                                                              |
| `less` | `/` 搜索，`n` 下一个匹配，`q` 退出                       | `less app.log` 分页查看大日志。                              |                                                              |
| `head` | `-n` 指定前 N 行                                         | `head -n 20 app.log` 查看文件开头。                          |                                                              |
| `tail` | `-n` 指定后 N 行，`-f` 持续监听                          | `tail -n 100 error.log` 查看最后 100 行；`tail -f app.log` 实时追日志。 |                                                              |
| `grep` | `-i` 忽略大小写，`-v` 反向匹配，`-r` 递归，`-n` 显示行号 | `grep -in "error" app.log` 查错误并显示行号；`grep -v "debug" app.log` 过滤 debug。 |                                                              |
| `awk`  | 按列提取、统计                                           | `ps aux \                                                    | awk '{print $2}' `提取 PID；`awk '{sum += $5} END {print sum}' file` 求和。 |
| `sed`  | 流式替换、删除、打印                                     | `sed -i 's/old_ip/new_ip/g' config.yml` 原地替换配置。       |                                                              |
| `wc`   | `-l` 行数，`-w` 单词数，`-c` 字节数                      | `grep "ERROR" app.log \                                      | wc -l` 统计错误行数。                                        |

**常见组合**

```bash
tail -f app.log | grep --line-buffered "ERROR"
ps aux | grep java | grep -v grep
grep -rn "timeout" /var/log/
```

### 3. 进程与服务管理

| 命令        | 核心参数                                       | 典型用法                                                     |                                        |                                     |
| ----------- | ---------------------------------------------- | ------------------------------------------------------------ | -------------------------------------- | ----------------------------------- |
| `ps`        | `aux` 或 `-ef` 查看进程快照                    | `ps aux \                                                    | grep nginx `查看 Nginx 进程；`ps -ef \ | grep java` 查看 Java 进程启动参数。 |
| `top`       | 动态查看 CPU、内存、负载                       | `top` 实时观察资源占用，按 `P` 按 CPU 排序，按 `M` 按内存排序。 |                                        |                                     |
| `kill`      | `-9` 强制杀死，`-15` 默认优雅终止              | `kill -15 <pid>` 先尝试正常终止；`kill -9 <pid>` 强制结束。  |                                        |                                     |
| `systemctl` | `start`、`stop`、`restart`、`status`、`enable` | `systemctl status nginx` 查看服务状态；`systemctl enable nginx` 设置开机自启。 |                                        |                                     |

**面试重点：`kill -9` 和普通 `kill` 的区别**

-   `kill` 默认发送 `SIGTERM`，进程可以捕获信号并做清理，例如释放资源、刷盘、关闭连接。
-   `kill -9` 发送 `SIGKILL`，进程无法捕获，系统直接终止；一般用于进程无响应时兜底，不应作为首选。

### 4. CPU、内存与磁盘排查

| 场景             | 命令              | 典型用法                                                     |
| ---------------- | ----------------- | ------------------------------------------------------------ |
| 查看内存         | `free`            | `free -h` 查看总内存、已用内存、可用内存。                   |
| 查看磁盘分区空间 | `df`              | `df -h` 查看各挂载点磁盘使用率。                             |
| 查看目录占用     | `du`              | `du -sh /var/log/` 汇总日志目录大小；`du -h --max-depth=1` 查看一级目录占用。 |
| 查看系统负载     | `uptime`、`top`   | `uptime` 查看 load average；`top` 定位 CPU 或内存占用高的进程。 |
| 查看 IO          | `iostat`、`iotop` | 生产环境常用于判断是否磁盘 IO 打满，部分系统需要额外安装。   |

**排查套路**

-   CPU 高：`top` 找进程 -> `ps -ef | grep <pid>` 看启动参数 -> 结合应用日志继续定位。
-   内存高：`free -h` 看整体内存 -> `top` 按内存排序 -> 判断是否缓存、泄漏或进程异常。
-   磁盘满：`df -h` 找满的挂载点 -> `du -h --max-depth=1` 逐层定位大目录 -> 清理日志或无用文件。

### 5. 权限与用户管理

Linux 权限按三类主体划分：所属用户 `User`、所属组 `Group`、其他用户 `Others`。权限数字分别是：读 `r=4`、写 `w=2`、执行 `x=1`。

| 命令     | 典型用法                                                     |
| -------- | ------------------------------------------------------------ |
| `chmod`  | `chmod 755 script.sh`：用户拥有读写执行权限，组和其他用户拥有读执行权限；`chmod +x script.sh` 添加执行权限。 |
| `chown`  | `chown -R www:www /var/www/html` 递归修改目录所属用户和用户组。 |
| `whoami` | 查看当前登录用户。                                           |
| `id`     | 查看当前用户 UID、GID 和所属组。                             |
| `sudo`   | 以管理员权限执行命令，依赖 sudoers 配置。                    |

**面试重点：`755` 和 `644`**

-   `755` 常用于目录或可执行脚本：所有者可读写执行，其他人可读和执行。
-   `644` 常用于普通文件：所有者可读写，其他人只读。
-   目录的 `x` 表示能进入目录；文件的 `x` 表示能执行文件。

### 6. 网络排查与远程连接

| 命令             | 核心参数                           | 典型用法                                                     |                                                    |
| ---------------- | ---------------------------------- | ------------------------------------------------------------ | -------------------------------------------------- |
| `ip`             | `addr` 查看地址，`route` 查看路由  | `ip addr` 查看网卡 IP；`ip route` 查看默认网关。             |                                                    |
| `ping`           | 测连通性                           | `ping 8.8.8.8` 判断网络是否可达。                            |                                                    |
| `curl`           | `-I` 只看响应头，`-v` 查看详细过程 | `curl -I <https://example.com`> 查看 HTTP 状态码；`curl -v` 排查连接和 TLS 问题。 |                                                    |
| `ss` / `netstat` | `-tulnp` 查看监听端口和进程        | `ss -tulnp \                                                 | grep 8080 `查看 8080 端口占用。新系统优先用` ss`。 |
| `ssh`            | `-p` 指定端口                      | `ssh root@192.168.1.100 -p 22` 登录远程服务器。              |                                                    |
| `scp`            | `-r` 递归传目录                    | `scp local.txt root@192.168.1.100:/tmp/` 上传文件；`scp -r dir/ host:/tmp/` 上传目录。 |                                                    |

**端口冲突排查**

```bash
ss -tulnp | grep 8080
lsof -i :8080
kill -15 <pid>
```

### 7. 管道、重定向与后台执行

| 符号 / 命令 | 作用               | 示例                                         |           |             |                                 |
| ----------- | ------------------ | -------------------------------------------- | --------- | ----------- | ------------------------------- |
| `           | `                  | 把前一个命令的输出交给后一个命令             | `ps aux \ | grep java \ | wc -l` 统计 Java 进程相关行数。 |
| `>`         | 覆盖写入文件       | `echo hello > a.txt`。                       |           |             |                                 |
| `>>`        | 追加写入文件       | `echo hello >> a.txt`。                      |           |             |                                 |
| `2>`        | 重定向错误输出     | `command 2> error.log`。                     |           |             |                                 |
| `&`         | 后台运行           | `sh start.sh &`。                            |           |             |                                 |
| `nohup`     | 退出终端后继续运行 | `nohup java -jar app.jar > app.log 2>&1 &`。 |           |             |                                 |

**面试重点：`>` 和 `>>` 的区别**

-   `>` 会覆盖原文件内容。
-   `>>` 会追加到文件末尾。
-   `2>&1` 表示把标准错误也重定向到标准输出，常和 `nohup` 一起使用。

### 8. 高频面试问法

-   如何排查 Linux 服务器磁盘满了？

    先用 `df -h` 找到使用率高的挂载点，再进入对应目录，用 `du -h --max-depth=1` 逐层定位大目录。常见大文件包括日志、临时文件、应用 dump 文件。确认文件不再被进程使用后再删除；如果删除后空间没有释放，可能是文件仍被进程占用，可用 `lsof | grep deleted` 排查。

-   如何查找某个端口被哪个进程占用？

    优先使用 `ss -tulnp | grep <port>`，也可以使用 `lsof -i :<port>`。查到 PID 后，用 `ps -ef | grep <pid>` 查看进程启动参数，再决定是否重启服务或终止进程。

-   如何实时查看日志并过滤关键字？

    使用 `tail -f app.log | grep --line-buffered "ERROR"`。如果只看最近一段日志，可以先用 `tail -n 200 app.log`；如果需要递归查历史日志，用 `grep -rn "ERROR" /var/log/`。

-   如何排查 CPU 使用率过高？

    先用 `top` 找到 CPU 占用高的进程 PID，再用 `ps -ef | grep <pid>` 查看进程信息。Java 服务还可以进一步用 `top -H -p <pid>` 找高 CPU 线程，再结合线程 dump 定位代码问题。

-   如何理解 Linux 文件权限？

    权限分为用户、用户组、其他用户三类，每类有读、写、执行三种权限。数字权限中 `r=4`、`w=2`、`x=1`，所以 `755` 表示所有者 `rwx`，组和其他用户 `r-x`；`644` 表示所有者可读写，其他用户只读。

### 9. 终端快捷键

| 快捷键     | 作用                   |
| ---------- | ---------------------- |
| `Tab`      | 自动补全命令或路径。   |
