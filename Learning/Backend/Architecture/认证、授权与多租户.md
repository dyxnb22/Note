# 认证、授权与多租户

认证回答“你是谁”，授权回答“你能做什么”，审计回答“你做过什么”。Cookie/Session/JWT 的协议层见 `CS/Network.md`；本页关注服务端如何把身份变成可验证的权限决策。

## 身份与会话

密码只存强哈希；第三方登录使用 OAuth/OIDC 的授权码流程；access token 应短期有效，refresh token 可撤销且需轮换。验证 token 的签名、issuer、audience、过期时间，不信任客户端提交的 user id 或 role。

## 授权模型

RBAC 用角色聚合权限，简单易管；ABAC 按用户/资源/环境属性判断，表达力强但难审计；资源级授权必须在服务端对“当前主体 + 目标资源 + 动作”判定。前端隐藏按钮不是权限控制。

## 多租户

租户隔离可在数据库、schema、行级字段或独立实例实现，隔离越强成本越高。无论选择何种模式，每次查询和缓存 key 都要带 tenant scope；后台任务、搜索索引、对象存储和日志也不可遗漏。用测试验证跨租户 ID 不可读取。

## 最小练习

为 SaaS 文档系统设计 `viewer/editor/admin` 权限表，并说明 API、缓存、ES 搜索如何执行租户隔离。
