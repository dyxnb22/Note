# 配置、密钥与环境治理

配置是随环境变化的行为参数，密钥是需要严格访问控制的凭据；二者都不应硬编码或提交 Git。配置还应区分启动时读取的静态项与可安全动态刷新的项。

## 分层与校验

默认值、环境变量、配置文件和部署平台注入可形成明确优先级。应用启动时解析成强类型配置并校验必填项、范围和依赖关系，避免运行半小时后才因 URL 为空失败。不要把整个环境变量表传遍业务层。

## 密钥生命周期

使用密钥管理服务/受控 secret store，按最小权限发放，定期轮换并记录访问审计。日志、异常、trace 和前端构建产物都可能泄漏 secret；redaction 是默认行为。轮换设计要支持新旧密钥短暂并存。

## 功能开关

Feature Flag 用于渐进发布、实验和紧急关闭，不是永久配置垃圾场。每个开关需要 owner、默认值、到期日、目标人群和回滚策略；权限相关开关尤其要在服务端求值。
