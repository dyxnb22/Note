# 业务项目能力地图

业务项目按共同骨架整理，具体实现和证据回到单独案例或源码审计。

| 项目 | 关键业务问题 | 可迁移能力 |
|---|---|---|
| 拼团交易平台 | 活动、人群、优惠、库存、支付、组队、结算、退款 | DDD、状态机、锁单、幂等、回调、补偿和对账 |
| 小型支付商城 | 商品、订单、支付、库存、MVC/DDD 对照 | 交易边界、支付状态、外部支付适配和部署 |
| 大营销/抽奖 | 策略、概率、奖品、库存、积分、返利 | 责任链、规则树、Redis 扣减、延迟队列、幂等发奖 |
| Lottery DDD 抽奖 | 策略库、活动、库存、抽奖和分布式协作 | 领域拆分、策略隔离、库存一致性和异步发奖 |
| IM Netty + JavaFX | 连接、会话、点对点/群组消息、断线重连 | 协议、连接生命周期、离线/重试、UI 与通信解耦 |
| 吉祥外卖 PLUS | 用户、商家、配送、运营、订单、支付、履约 | 多角色视图、领域边界、履约状态、任务补偿 |

## 通用链路

```text
用户动作/回调/定时任务
  → 接口协议与鉴权
  → 应用校验与编排
  → 领域状态/规则/聚合
  → 仓储、缓存、消息、支付适配器
  → 回执、异步事件、对账和补偿
```

## 必须显式建模的状态

订单/组队、支付、库存/名额、权益、消息分别记录生命周期。试算、资源预占、最终扣减、发放和核销是不同事实；所有回调、消息、任务和重试默认可能重复。

## 抽奖项目的高并发骨架

```text
参与订单/额度 → 责任链 → 抽奖算法 → 规则树 → Redis 原子扣减
                                      ↓
                               发奖 MQ/延迟队列
                                      ↓
                            MySQL 账面库存/对账
```

Redis 负责瞬时并发，延迟队列和 MQ 负责削峰与重试，数据库负责最终账面；库存、积分、支付和发奖通过唯一业务号串联。
