## 可迁移链路

### 拼团交易平台

`Controller → 市场试算 → 规则/策略 → 锁单 → 结算/退款 → Job/MQ/通知`。重点是策略工厂、活动/标签/库存规则、交易状态、超时退款和 DCC 动态配置。

### 吉祥外卖 PLUS

API 负责 DTO，Domain 按用户、商品、订单、支付、配送、商户和运营分域，Infrastructure 用 Repository/DAO/PO/MyBatis 适配数据库。下单、支付、配送是相互依赖但独立的状态链路；第三方支付/配送以端口存在，不能当作已联调。

### 鱼塘/钓鱼佬

用户、鱼、投票、钓鱼、积分和统计通过 API/Trigger 进入 Redis/Redisson；适合观察高频状态、延迟队列、锁、集合、布隆过滤器和双语言边界。微信认证和 Redis 外部能力仍需联调。

### WaLiSSH

`API/Trigger → App → Case(ReAct/意图) → Domain(Agent/SSH) → Infrastructure(JSch/SFTP/LLM)`。先验证连接，再加入命令、Agent 和 SFTP；服务端统一承载凭证、权限、危险命令拦截和审计。

### WaLiCode

在 WaLiSSH 的 Agent/MCP 基础上增加项目上下文、任务拆解、记忆、代码编辑、本地/SSH 执行、构建验证和子 Agent。模型负责决策，服务端负责权限确认、执行顺序和结果验证。

### 手写 MyBatis

```text
XML/注解 → Configuration/MapperRegistry → MapperProxy
→ SqlSession/Executor → 参数处理 → JDBC
→ 结果映射 → 缓存/插件 → Spring 整合
```

20 个步骤逐步加入代理、配置解析、数据源、执行器、动态 SQL、插件、一级/二级缓存和 Spring 注册。重点是组件职责和调用链，不是复制实现代码。

## 统一风险

- 测试插件可能默认跳过测试，必须核对 Surefire 配置和 `Tests run` 数量。
- 构建成功只证明编译/打包；数据库、Redis、MQ、模型、支付、SSH 和微信需要独立验证。
- 端口/适配器可以证明设计边界存在，不能证明第三方实现已经可用。
- 项目文档、源码、配置、测试和部署脚本要互相校验，发现差异时记录，不用猜测补齐。
