## Learning Goals

- What should be understood after this module?

## Outputs

- Personal notes
- Reproduction baseline
- From-scratch implementation
- Experiments or ablations
- Final report
- One clear project artifact that can be described in one sentence

## Checklist

- [ ] Read the reference chapter and collect links
- [ ] Write original notes in `notes.md`
- [ ] Reproduce a baseline in `reproduce/`
- [ ] Implement the core idea in `from_scratch/`
- [ ] Run ablations or diagnostics in `experiments/`
- [ ] Summarize results in `report.md`
- [ ] Confirm the module has a standalone project identity

## Suggested Questions

- What problem does this method solve?
- What assumptions does it make?
- Which parts are easy to reproduce with a library but harder to understand from scratch?
- What breaks when hyperparameters or architecture choices change?
- What did the experiments reveal that reading alone would not?
