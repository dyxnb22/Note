# 06 生产级 LLM 系统

## 生产环境关心什么

做 Demo 时关注“模型能不能回答”，做生产系统时关注：

- 超时怎么办。
- API 报错怎么办。
- 成本是否可控。
- 是否有日志和 trace。
- 是否能复现一次坏回答。
- 用户输入是否存在 prompt injection。
- 工具调用是否有权限边界。

## LLM 网关常见能力

- 统一模型调用接口。
- 重试与超时。
- 缓存相同请求。
- 限流。
- 日志与成本统计。
- 敏感词或隐私数据过滤。
- 模型降级策略。

## 项目说明

`project/llm_gateway.py` 是一个标准库 HTTP 服务，演示最小 LLM 网关：

- `POST /chat` 调用模型或 mock。
- 内存缓存。
- 简单限流。
- 请求日志。
- 超时与错误处理。

运行：

```bash
cd 06_production_llm_system/project
python3 llm_gateway.py
curl -X POST http://127.0.0.1:8088/chat -d '{"message":"解释 RAG"}'
```
