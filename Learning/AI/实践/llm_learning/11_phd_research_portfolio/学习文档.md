# 11 PhD 申请研究作品集

全奖 PhD 申请看重的不是“学过很多名词”，而是你是否有研究潜力。你需要把学习成果转成可证明的材料。

## 申请前你最好具备

- 扎实 ML/DL/LLM 基础。
- 2-3 个高质量复现项目。
- 1 个有新意的 research project。
- 1 篇英文 technical report 或 workshop-style paper。
- GitHub repo 清楚、可复现、有实验日志。
- 能写出 research statement。
- 能联系潜在导师并说明 fit。

## 推荐作品集结构

### Project 1：From-Scratch LM

目标：
- 实现 tokenizer、Transformer、训练循环、评测。
- 对齐 Stanford CS336 的训练思路。

价值：
- 证明你不是只会调 API。

### Project 2：Reasoning / Alignment

目标：
- 复现 DPO 或 reasoning distillation 小实验。
- 比较 SFT、DPO、CoT 数据的效果。

价值：
- 对齐当前 reasoning model 研究热点。

### Project 3：Evaluation / Safety / Agent

目标：
- 做一个新的评测集或评测方法。
- 关注 Agent tool-use、prompt injection、CoT monitorability 或成本鲁棒性。

价值：
- 这类方向更适合个人资源有限时做出独立贡献。

### Project 4：Interpretability

目标：
- 用小模型做 activation patching 或 circuit 分析。
- 写出可视化和结论。

价值：
- 体现研究深度和可解释性兴趣。

## 12 个月路线

1-2 月：
- 补数学、深度学习、Transformer。
- 完成 from-scratch toy LM。

3-4 月：
- 跟 Stanford CS336 做一遍核心作业。
- 训练一个小模型，写训练报告。

5-6 月：
- 做 RAG/Agent/评测项目。
- 写系统性技术报告。

7-8 月：
- 复现一篇 alignment/reasoning 论文。
- 做 ablation。

9-10 月：
- 形成自己的研究问题。
- 写 mini paper。

11-12 月：
- 打磨 GitHub、CV、research statement。
- 联系导师，准备申请。

## 项目说明

`project/research_idea_scorer.py` 帮你给研究 idea 打分。它不会替你做研究，但能提醒你从 novelty、feasibility、evaluation、PhD fit 角度筛选方向。

运行：

```bash
cd 11_phd_research_portfolio/project
python3 research_idea_scorer.py
```
