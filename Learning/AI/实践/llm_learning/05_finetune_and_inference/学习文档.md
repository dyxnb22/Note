# 05 微调与推理

## 必学模型原理

- Tokenizer：把文本切成 token，模型实际处理的是 token id。
- Transformer：由 Attention、MLP、残差连接、归一化等组成。
- Attention：让当前位置关注上下文中相关 token。
- Pretrain：用海量文本学习通用语言能力。
- SFT：用指令数据让模型更会按要求回答。
- RLHF/DPO：进一步对齐人类偏好。
- LoRA：冻结原模型，只训练低秩适配矩阵。

## RAG 与微调的边界

- 业务知识更新快：RAG。
- 输出格式稳定、风格统一：微调。
- 模型不会某种任务模式：微调。
- 模型不知道某份新文档：RAG。

## 推理优化关键词

- KV Cache：缓存历史 token 的 key/value，减少重复计算。
- Batch：多个请求一起推理，提高吞吐。
- Quantization：降低权重精度，减少显存和内存。
- Speculative Decoding：用小模型草稿加速大模型生成。
- Streaming：边生成边返回，降低用户感知延迟。

## 项目说明

`project/prepare_sft_dataset.py` 不训练大模型，而是演示岗位里常见的数据准备工作：

- 把问答样例转换成 chat SFT JSONL。
- 检查字段是否完整。
- 估算 token 数。
- 生成一份 LoRA 训练配置草案。

运行：

```bash
cd 05_finetune_and_inference/project
python3 prepare_sft_dataset.py
```
