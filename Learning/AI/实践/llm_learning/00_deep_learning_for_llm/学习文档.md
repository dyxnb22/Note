# 00 深度学习到 LLM 基础

这条路线解决的问题是：你不只是会用大模型 API，而是知道大模型为什么能训练出来、训练时发生了什么、推理时为什么贵、为什么 DeepSeek 这样的模型要做 MoE/MLA/RL。

## 阶段 A：数学与神经网络直觉

目标：
- 理解向量、矩阵乘法、点积、范数。
- 理解 softmax、交叉熵、梯度下降。
- 理解反向传播是在计算“每个参数应该往哪个方向改”。

你需要会解释：
- 为什么模型输出的是概率分布。
- 为什么 loss 越小代表预测越接近目标。
- 为什么学习率太大会震荡，太小会训练慢。

练习：
- 跑通 `project/tiny_next_token_model.py`。
- 改训练语料，观察 loss 和预测结果变化。

## 阶段 B：从神经网络到语言模型

目标：
- 理解语言模型的任务：给定前文，预测下一个 token。
- 理解 embedding 层：把 token id 变成向量。
- 理解分类头：把隐藏向量映射成词表概率。

关键公式：

```text
input token -> embedding -> neural network -> logits -> softmax -> next token probability
```

面试表达：

“LLM 的训练目标通常是 next-token prediction。模型看到前面的 token，输出词表上每个 token 的概率，通过交叉熵比较预测分布和真实下一个 token，然后反向传播更新参数。”

## 阶段 C：Transformer 结构

目标：
- 理解 self-attention 的 Q/K/V。
- 理解 multi-head attention。
- 理解 FFN、Residual、LayerNorm。
- 理解 causal mask。

对应目录：
- `00_transformer_foundation/`

## 阶段 D：训练流程

目标：
- 预训练：海量通用语料上做 next-token prediction。
- SFT：用指令数据让模型学会按人类要求回答。
- Preference Alignment：RLHF/DPO/GRPO 等方法让模型更符合偏好。
- Distillation：用大模型生成数据训练小模型。

你需要能区分：
- 预训练让模型获得通用语言和知识能力。
- SFT 让模型更会“听指令”。
- RL/偏好优化让模型更符合推理、安全、格式等偏好。
- 蒸馏让小模型学习大模型的输出模式。

## 阶段 E：推理系统

目标：
- 理解 KV Cache。
- 理解 batch、prefill、decode。
- 理解量化和显存估算。
- 理解为什么长上下文很贵。

面试高频：
- KV Cache 为什么加速生成？
- INT4/INT8 量化为什么省显存？
- 为什么 batch 能提高吞吐但可能增加延迟？

## 阶段 F：开源模型阅读

先读小模型：
- GPT-2 风格 Decoder-only。
- Llama/Qwen 风格现代 Decoder-only。

再读高级模型：
- MoE：每个 token 只激活部分专家，降低每 token 计算成本。
- MLA：压缩注意力中的 KV 表示，降低推理缓存压力。
- Reasoning RL：通过奖励信号强化推理过程。

最后进入：
- `07_open_llm_deepseek_study/`

## 最终能力标准

学完这条线，你应该能做到：

- 手写解释 next-token prediction。
- 看懂一个简化 Transformer 的 forward 过程。
- 看懂 LoRA 微调数据和训练参数。
- 能解释 DeepSeek-V3 为什么用 MoE/MLA。
- 能解释 DeepSeek-R1 为什么强调 RL、冷启动数据和蒸馏。
- 面试时能把“模型结构、训练、推理、应用”串成完整闭环。
