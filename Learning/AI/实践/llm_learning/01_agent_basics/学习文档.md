# 01 Agent 基础

## 你需要掌握什么

Agent 的本质不是“模型自己执行任务”，而是一个循环：

1. 用户提出目标。
2. LLM 判断下一步要不要调用工具。
3. 程序执行工具并返回观察结果。
4. LLM 根据观察结果继续推理或给最终答案。

这类模式常被称为 ReAct：Reasoning + Acting。

## 必学概念

- Prompt：给模型的任务说明与约束。
- Tool：由程序提供的可执行能力，例如计算、查询、写文件。
- Observation：工具执行后的结果。
- Memory：短期上下文或长期资料。
- Guardrail：权限、校验、确认机制。

## 找工作时怎么讲

可以这样表达：

“我实现过一个简单 Agent 框架，模型只负责选择工具和生成参数，真正的工具执行由后端完成。这样可以控制权限、记录日志，并且避免模型直接执行危险操作。”

## 常见坑

- 工具结果没有结构化，模型很难继续推理。
- 把敏感操作直接交给模型，没有确认步骤。
- Prompt 写得很长但没有评测，改动后不知道效果变好还是变差。
- 误以为 Agent 越多越好，其实简单任务单 Agent 更稳定。

## 项目说明

`project/simple_agent.py` 是一个命令行 Agent。它支持：

- 计算表达式。
- 查询当前时间。
- 保存一条本地笔记。
- 在没有 API key 时使用 mock 模型输出。

运行：

```bash
cd 01_agent_basics/project
python3 simple_agent.py "帮我计算 12 * 8 + 6"
```
