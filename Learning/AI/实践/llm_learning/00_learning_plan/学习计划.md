# 大模型学习计划

目标：从“能独立开发 Agent/RAG 应用”逐步提升到“理解大模型训练、推理、评测、部署与工程可靠性”，达到可以投递大模型应用工程师、AI Agent 工程师、RAG 工程师、LLM 平台工程师初中级岗位的程度。

## 阶段 -1：深度学习到 LLM 基础

周期：2-3 周

目标：
- 理解 softmax、cross entropy、梯度下降。
- 理解语言模型的 next-token prediction 训练目标。
- 理解 token -> embedding -> logits -> probability 的基本链路。
- 为 Transformer、SFT、RL、推理优化打基础。

产出：
- 完成 `00_deep_learning_for_llm/project/tiny_next_token_model.py`。
- 能解释 loss、梯度、学习率、过拟合。
- 能说明大模型训练为什么本质上是预测下一个 token。

## 阶段 0：Transformer 基础

周期：2-3 周

目标：
- 理解 token、embedding、矩阵乘法、softmax。
- 理解 self-attention 的 Q/K/V 计算过程。
- 理解 Transformer Block、causal mask、Decoder-only 生成。
- 能用自己的话解释为什么 Transformer 适合大模型。

产出：
- 完成 `00_transformer_foundation/project/self_attention_demo.py`。
- 能手写解释 `Attention(Q, K, V) = softmax(QK^T / sqrt(d_k))V`。
- 能讲清楚 Decoder-only LLM 为什么是“预测下一个 token”。

## 阶段 1：Agent 入门基础

周期：1-2 周

目标：
- 理解 LLM、Prompt、上下文窗口、工具调用、Agent 循环。
- 能写一个命令行 Agent，支持调用本地工具。
- 明白 Agent 不是“模型自动万能”，而是“模型负责决策，程序负责执行”。

产出：
- 完成 `01_agent_basics/project/simple_agent.py`。
- 能解释 ReAct、工具 schema、函数调用、幻觉与权限控制。

## 阶段 2：RAG 知识库

周期：2-3 周

目标：
- 理解 embedding、chunk、召回、重排、上下文拼接。
- 能做一个本地 Markdown/TXT 知识库问答。
- 知道 RAG 失败常见原因：切分差、召回差、上下文污染、问题改写不足、评测缺失。

产出：
- 完成 `02_rag_knowledge_base/project/mini_rag.py`。
- 准备一份自己的学习笔记知识库并能查询。

## 阶段 3：工具调用与工作流

周期：2 周

目标：
- 理解单 Agent、Planner-Executor、状态机、多 Agent 的差异。
- 能设计“先计划、后确认、再执行”的 Agent 工作流。
- 学会把不可逆操作放到确认节点。

产出：
- 完成 `03_tool_calling_workflow/project/workflow_agent.py`。
- 能画出 Agent 执行链路：输入 -> 计划 -> 工具 -> 观察 -> 纠错 -> 输出。

## 阶段 4：评测与调优

周期：2 周

目标：
- 建立大模型项目的评测意识。
- 理解离线评测、人工评测、LLM-as-judge、回归测试。
- 能用测试集比较不同 prompt 或不同模型。

产出：
- 完成 `04_llm_evaluation/project/prompt_eval.py`。
- 每个 Agent/RAG 项目都能配一套最小评测集。

## 阶段 5：微调、推理与模型原理

周期：4-6 周

目标：
- 理解 Transformer、Attention、Tokenization、预训练、SFT、RLHF/DPO、LoRA。
- 能准备 SFT 数据，知道何时该 RAG、何时该微调。
- 理解 KV Cache、量化、吞吐、延迟、显存估算。

产出：
- 完成 `05_finetune_and_inference/project/prepare_sft_dataset.py`。
- 能解释 LoRA 为什么省显存、量化为什么影响精度、KV Cache 为什么提升生成速度。

## 阶段 6：生产级 LLM 系统

周期：3-5 周

目标：
- 理解 LLM 网关、重试、限流、缓存、日志、成本控制、安全策略。
- 能把模型能力包装成稳定 API。
- 能处理超时、失败、降级、审计、提示词注入。

产出：
- 完成 `06_production_llm_system/project/llm_gateway.py`。
- 能设计一个公司内部知识库 Agent 的生产架构。

## 阶段 7：开源 LLM 与 DeepSeek 级模型理解

周期：4-8 周

目标：
- 读懂现代 Decoder-only 模型的关键组件。
- 理解 MoE、MLA、KV Cache、量化、蒸馏。
- 能解释 DeepSeek-V3 的 MoE/MLA/FP8/MTP 取舍。
- 能解释 DeepSeek-R1 的 RL、冷启动数据、蒸馏模型路线。

产出：
- 完成 `07_open_llm_deepseek_study/project/architecture_budget_estimator.py`。
- 能读 DeepSeek-V3/R1 官方 README 和技术报告摘要。
- 能在 Mac 上运行一个小型量化开源模型或蒸馏模型。

## 阶段 8：研究方法与论文复现

目标：
- 理解 SFT、RLHF、DPO、distillation、reasoning RL。

## 每周节奏建议

- 周一到周三：学概念，看代码，补基础。
- 周四到周五：实现项目，写测试样例。
- 周六：总结文档，补面试表达。
- 周日：复盘，整理简历项目描述。
