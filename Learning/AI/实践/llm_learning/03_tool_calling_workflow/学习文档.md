# 03 工具调用与工作流

## 为什么要从 Agent 升级到 Workflow

简单 Agent 适合探索式任务，但生产环境更关心稳定性。很多业务流程应该有明确节点：

- 计划节点：模型提出方案。
- 审核节点：程序或用户确认。
- 执行节点：调用工具。
- 校验节点：检查结果。
- 归档节点：记录日志。

这类设计可以减少“模型直接乱做事”的风险。

## 常见架构

- 单 Agent：实现简单，但复杂任务容易发散。
- Planner-Executor：一个模型负责计划，一个执行器负责工具。
- State Machine：每一步有明确状态，适合生产。
- Multi-Agent：适合角色边界清晰的复杂任务，但调试成本高。

## 项目说明

`project/workflow_agent.py` 演示一个确认式工作流：

1. 根据用户目标生成计划。
2. 风险检查。
3. 只执行允许的本地动作。
4. 输出执行报告。

运行：

```bash
cd 03_tool_calling_workflow/project
python3 workflow_agent.py "整理今天的大模型学习任务"
```
