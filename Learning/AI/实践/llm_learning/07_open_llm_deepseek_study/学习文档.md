# 07 开源 LLM 与 DeepSeek 学习阶段

这个阶段不是让你在个人 Mac 上训练 DeepSeek 级别模型。真实训练需要大规模 GPU 集群。你的目标应该是：

- 看懂架构为什么这样设计。
- 能运行蒸馏小模型或 API。
- 能读懂论文和关键代码。
- 能在面试中解释训练、推理、成本与工程取舍。

## 阶段 1：先读现代 Decoder-only

先掌握：
- RMSNorm
- RoPE
- SwiGLU
- GQA/MQA
- Causal Self-Attention
- KV Cache

建议顺序：
1. GPT-2 简化实现。
2. Llama/Qwen 类架构。
3. 再看 MoE 和 MLA。

## 阶段 2：理解 MoE

MoE，Mixture of Experts，核心思想是：

每个 token 不一定经过全部 FFN 参数，而是由 router 选择少数专家参与计算。

好处：
- 总参数可以很大。
- 每个 token 激活参数相对较少。
- 训练和推理成本更可控。

难点：
- 路由负载均衡。
- 专家并行通信。
- 训练稳定性。
- 部署复杂度。

DeepSeek-V3/R1 公开信息里，671B total parameters、37B activated parameters per token 是理解 MoE 的重要数字。

## 阶段 3：理解 MLA

MLA，Multi-head Latent Attention，可以粗略理解为对 KV cache 做更高效的低维潜在表示，目标是降低长上下文推理时的缓存压力。

你需要知道：
- 普通 attention 在长上下文下 KV cache 很占显存。
- GQA/MQA 是降低 KV 头数量。
- MLA 是更进一步的注意力/KV 压缩设计。

## 阶段 4：理解 DeepSeek-V3

重点不是背参数，而是理解工程取舍：

- MoE：大总参数，小激活参数。
- MLA：降低推理缓存成本。
- FP8 mixed precision：降低训练成本。
- Multi-Token Prediction：训练时预测多个未来 token，增强训练信号。
- 大规模高质量数据与训练系统协同。

读法：
1. 先看 README 的 Model Summary。
2. 再看技术报告摘要和架构图。
3. 最后看 inference 代码里的模型配置、attention、MoE 层。

## 阶段 5：理解 DeepSeek-R1

R1 重点是 reasoning：

- R1-Zero：直接在 base model 上做大规模 RL，观察到推理行为自然涌现。
- R1：加入冷启动数据，改善可读性、语言混杂等问题。
- Distill：用 R1 生成的数据蒸馏到 Qwen/Llama 等较小 dense 模型。

你需要会讲：
- 为什么 reasoning model 不只是普通 chat model。
- 为什么纯 RL 会出现可读性和语言混杂问题。
- 为什么蒸馏能让小模型获得一部分推理模式。

## 阶段 6：Mac 上能做什么

你的 Mac 适合做：
- 跑 1.5B/7B 量化模型。
- 用 Ollama/llama.cpp/MLX 学推理。
- 读模型配置和 forward 代码。
- 做 LoRA 小实验。
- 做 RAG/Agent 应用。

不适合做：
- 从零预训练百亿级模型。
- 复现 DeepSeek-V3/R1 全量训练。
- 在本地完整跑 671B 模型。

## 项目说明

`project/architecture_budget_estimator.py` 用非常简化的估算，帮助你理解 dense 模型和 MoE 模型的参数激活差异，以及 KV cache 为什么会随上下文长度增长。

运行：

```bash
cd 07_open_llm_deepseek_study/project
python3 architecture_budget_estimator.py
```
