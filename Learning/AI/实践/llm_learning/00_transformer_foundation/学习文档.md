# 00 Transformer 基础

如果你想真正学大模型，建议从 Transformer 学起，但不要一上来就啃论文公式。更好的顺序是：

1. 先理解 token、向量、矩阵乘法。
2. 再理解 attention 在做什么。
3. 然后理解 Transformer Block。
4. 最后再学预训练、微调、推理优化和 Agent 应用。

## 第一层：大模型到底在做什么

大模型不是直接理解文字。它的基本过程是：

1. 文本被 tokenizer 切成 token。
2. token 被映射成向量。
3. Transformer 反复处理这些向量。
4. 模型预测下一个 token 的概率。
5. 系统按概率采样或选择 token，拼成最终回答。

一句话：大模型本质上是“根据上下文预测下一个 token”的神经网络。

## 第二层：为什么需要 Attention

如果输入是：

```text
我 喜欢 学习 大模型
```

模型处理“大模型”这个 token 时，需要知道前面的“学习”很重要，也可能需要知道“我喜欢”提供了语义背景。

Attention 的作用就是：让每个 token 根据当前任务，动态关注上下文里更重要的 token。

## 第三层：Q、K、V 怎么理解

Attention 里每个 token 会生成三种向量：

- Q：Query，表示“我现在想找什么信息”。
- K：Key，表示“我能提供什么索引信息”。
- V：Value，表示“我真正携带的内容”。

计算流程：

1. 用 Q 和所有 K 做相似度。
2. 相似度经过 softmax 变成权重。
3. 用权重加权求和所有 V。
4. 得到当前 token 聚合后的新表示。

公式：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k))V
```

## 第四层：Transformer Block 由什么组成

一个标准 Transformer Block 通常包含：

- Self-Attention：让 token 互相看见。
- Feed Forward Network：对每个 token 的表示做非线性变换。
- Residual Connection：保留原始信息，缓解深层训练困难。
- LayerNorm：稳定训练。

大模型就是把很多个 Block 堆起来。

## 第五层：Decoder-only LLM

ChatGPT、DeepSeek、Llama 这类文本生成模型通常是 Decoder-only 架构。

它们生成时有一个关键限制：当前位置只能看见自己和前面的 token，不能看未来 token。这个机制叫 causal mask。

例如生成：

```text
我 喜欢 学习
```

预测“学习”时，模型可以看“我 喜欢”，但不能偷看后面的答案。

## 第六层：从 Transformer 到大模型工程

学完 Transformer 后，再往下走：

1. Tokenizer：为什么中文、英文、代码的 token 成本不同。
2. Pretraining：为什么海量文本能训练通用能力。
3. SFT：为什么指令微调能让模型更听话。
4. RLHF/DPO：为什么需要偏好对齐。
5. RAG：为什么外部知识不应该全靠模型记忆。
6. Agent：为什么模型需要工具才能完成真实任务。
7. Evaluation：如何证明模型应用真的变好了。
8. Serving：如何控制成本、延迟、并发和稳定性。

## 面试表达

可以这样说：

“Transformer 的核心是 self-attention。每个 token 会生成 Q、K、V，Q 和 K 计算相关性，softmax 得到权重，再对 V 加权求和。这样模型可以动态关注上下文中最相关的信息。Decoder-only 大模型在生成时使用 causal mask，只能看当前位置之前的 token，因此可以自回归地预测下一个 token。”
