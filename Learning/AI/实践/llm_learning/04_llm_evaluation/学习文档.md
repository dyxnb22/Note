# 04 LLM 评测

## 为什么评测很重要

大模型应用最容易出现的问题是“感觉变好了”。没有评测集时，Prompt、模型、检索策略、工具描述的改动都无法比较。

评测至少分三层：

- 单轮问答：答案是否正确。
- RAG：是否召回到正确文档，答案是否引用依据。
- Agent：是否选择正确工具，是否完成任务，是否避免危险动作。

## 常用指标

- Exact Match：完全匹配，适合确定性答案。
- Keyword Hit：关键点覆盖。
- Retrieval Recall@K：正确文档是否进入前 K。
- LLM-as-judge：让另一个模型按 rubric 打分。
- Human Review：关键业务场景必须人工抽检。

## 项目说明

`project/prompt_eval.py` 用一个小测试集比较两种 prompt 风格。默认使用 mock answer，设置 API key 后可真实调用模型。

运行：

```bash
cd 04_llm_evaluation/project
python3 prompt_eval.py
```
