# 10 可解释性、评测与安全研究

想申请 PhD，评测和可解释性是非常值得投入的方向。它们不一定需要最大 GPU，但需要严谨的问题定义、实验设计和分析能力。

## 方向 1：LLM Evaluation

研究问题：

- 现有 benchmark 是否污染？
- Agent 是否真的完成任务，还是只会生成看似合理的日志？
- LLM-as-judge 是否稳定？
- 多轮对话和工具调用如何评测？
- 推理模型是否存在 hidden failure mode？

你需要掌握：

- 数据集构造。
- 指标设计。
- 人工标注一致性。
- 统计显著性。
- 错误分类 taxonomy。

## 方向 2：Mechanistic Interpretability

目标是理解模型内部机制，而不只是看输入输出。

核心概念：

- Activation。
- Attention head。
- MLP neuron。
- Circuit。
- Activation patching。
- Sparse Autoencoder。

入门顺序：

1. 先看小 Transformer。
2. 再用 TransformerLens 分析 GPT-2 small。
3. 做简单的 activation patching。
4. 尝试解释一个具体行为，例如 induction head。

## 方向 3：Safety and Robustness

研究问题：

- Prompt injection 如何检测？
- Agent 工具调用如何防越权？
- Reasoning model 是否会 reward hacking？
- CoT 可监控性是否会随训练方式变化？

## 项目说明

`project/eval_significance.py` 演示一个非常小的评测统计流程：比较两个 prompt 在同一测试集上的胜率，并用 bootstrap 估计置信区间。

运行：

```bash
cd 10_interpretability_and_evaluation/project
python3 eval_significance.py
```
