# 大模型八股

## 1. Transformer 为什么重要？

Transformer 用自注意力机制建模 token 之间的关系，相比 RNN 更适合并行训练，也更容易扩展到大规模参数和数据。现代大语言模型大多基于 Decoder-only Transformer。

## 2. Attention 的核心公式是什么？

常见表述是 `softmax(QK^T / sqrt(d_k))V`。Q 表示当前 token 查询什么，K 表示每个 token 提供什么索引，V 表示真正聚合的信息。

## 3. 为什么要除以 `sqrt(d_k)`？

点积维度越高，数值方差越大，softmax 容易过度尖锐，导致梯度不稳定。缩放可以让训练更稳定。

## 4. Decoder-only、Encoder-only、Encoder-Decoder 有什么区别？

Encoder-only 适合理解类任务，如分类和匹配。Decoder-only 适合自回归生成，是主流 LLM 架构。Encoder-Decoder 常用于翻译、摘要等输入输出都较长的任务。

## 5. 什么是 Tokenizer？

Tokenizer 把文本转成 token id。模型不直接理解字符，而是处理 token 序列。不同 tokenizer 会影响上下文长度、成本和多语言效果。

## 6. 上下文窗口越长越好吗？

不一定。长上下文会增加成本和延迟，也可能让模型更难关注关键内容。工程上常结合 RAG、摘要和记忆管理。

## 7. 什么是 KV Cache？

生成第 N 个 token 时，前面 token 的 key/value 已经算过。KV Cache 缓存这些中间结果，避免重复计算，从而加速自回归推理。

## 8. 什么是 RAG？

RAG 是先从外部知识库检索相关内容，再把内容作为上下文交给模型回答。它适合知识更新快、需要引用来源、需要降低幻觉的场景。

## 9. RAG 常见失败原因？

文档切分不合理、embedding 模型不匹配、召回结果不准、上下文过长、没有重排、Prompt 没约束引用、缺少评测集。

## 10. Embedding 是什么？

Embedding 是把文本映射成向量，语义相近的文本向量距离更近。RAG 常用 embedding 做相似度检索。

## 11. 向量数据库解决什么问题？

它负责高效存储和检索 embedding，支持近似最近邻搜索、过滤条件、元数据管理。常见选择包括 FAISS、Milvus、Qdrant、pgvector。

## 12. Agent 和普通 Chatbot 的区别？

普通 Chatbot 主要生成文本。Agent 会根据目标进行规划、调用工具、观察结果并继续决策。Agent 的关键是工具执行和控制循环。

## 13. Tool Calling 的本质是什么？

模型输出结构化调用意图，例如函数名和参数；程序解析后执行真实函数。模型不应该直接拥有系统权限。

## 14. ReAct 是什么？

ReAct 是 Reasoning + Acting。模型一边推理，一边选择动作，再根据工具观察结果继续推理。

## 15. 为什么 Agent 要做权限控制？

因为模型可能误判、被提示词注入、生成错误参数。删除、支付、发消息、写数据库等动作都应该确认、审计、可回滚。

## 16. Prompt Engineering 重要吗？

重要，但不能只靠 Prompt。生产系统需要工具设计、检索质量、评测、日志、权限和降级策略。

## 17. 什么是 SFT？

SFT 是监督微调，用高质量指令-回答数据训练模型，让模型更符合任务格式和回答风格。

## 18. 什么是 RLHF？

RLHF 用人类偏好训练奖励模型，再优化语言模型输出，使回答更符合人类偏好。但流程复杂、成本较高。

## 19. 什么是 DPO？

DPO 直接使用偏好数据优化模型，不需要显式训练奖励模型，工程实现比传统 RLHF 更简单。

## 20. 什么是 LoRA？

LoRA 冻结原模型权重，只训练低秩矩阵适配任务。它显著减少可训练参数和优化器状态，降低微调成本。

## 21. 什么时候用微调，什么时候用 RAG？

知识更新和事实查询优先 RAG。输出风格、任务格式、领域表达习惯可以考虑微调。很多生产系统会同时使用 RAG 和轻量微调。

## 22. 什么是量化？

量化把模型权重从 FP16/BF16 降到 INT8、INT4 等更低精度，以减少显存和提升推理速度，但可能损失精度。

## 23. Temperature 是什么？

Temperature 控制采样随机性。低 temperature 更稳定，高 temperature 更多样。严肃问答、代码、抽取任务通常设置较低。

## 24. Top-p 是什么？

Top-p 又叫 nucleus sampling，只从累计概率达到 p 的 token 集合中采样，用于控制生成多样性。

## 25. 幻觉是什么？

模型生成看似合理但事实错误的内容。减少幻觉的方法包括 RAG、引用来源、拒答策略、结构化输出、评测和人工审核。

## 26. LLM 项目如何评测？

先构建代表性测试集，再分别评估正确性、引用依据、格式遵守、工具选择、延迟、成本和安全性。不能只凭主观感觉。

## 27. LLM-as-judge 有什么风险？

评审模型也会偏见和误判。最好配合明确 rubric、少量人工抽检，以及固定测试集回归。

## 28. 生产级 LLM 网关需要什么能力？

统一模型接口、鉴权、限流、重试、超时、缓存、日志、成本统计、trace、降级、安全过滤和审计。

## 29. Prompt Injection 是什么？

用户通过输入诱导模型忽略系统指令、泄露隐私或执行危险动作。防护要靠权限隔离、输入过滤、工具确认和最小权限。

## 30. 面试如何介绍一个 RAG + Agent 项目？

可以按“业务目标 -> 文档处理 -> 检索策略 -> Agent 工具 -> 评测指标 -> 生产保护”讲。重点说明为什么这样设计、失败怎么处理、如何证明效果变好。
