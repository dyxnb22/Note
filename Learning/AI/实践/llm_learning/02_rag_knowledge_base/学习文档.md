# 02 RAG 知识库

## RAG 是什么

RAG，全称 Retrieval-Augmented Generation，意思是“先检索，再生成”。它适合回答公司文档、项目知识、个人笔记、业务规则这类不适合全部塞进模型参数里的知识。

核心流程：

1. 文档切分成 chunk。
2. 为每个 chunk 建索引。
3. 用户提问时召回相关 chunk。
4. 把 chunk 作为上下文交给模型回答。
5. 返回答案时最好带引用来源。

## RAG 与微调怎么选

- 知识经常变化：优先 RAG。
- 需要回答可追溯：优先 RAG。
- 需要改变模型表达风格或固定任务格式：可以考虑微调。
- 需要学习新事实：不要优先微调，通常 RAG 更便宜、更可控。

## 项目说明

`project/mini_rag.py` 使用标准库实现一个迷你 RAG：

- 从 `docs/` 读取 `.txt` 和 `.md`。
- 用词频余弦相似度做检索。
- 没有 API key 时直接返回最相关上下文。
- 有 API key 时把上下文交给模型生成答案。

运行：

```bash
cd 02_rag_knowledge_base/project
python3 mini_rag.py --init
python3 mini_rag.py "Agent 为什么需要工具调用"
```
