# 09 对齐、偏好优化与推理模型研究

这是当前 LLM 研究最重要的方向之一。你需要理解模型不只是“生成更流畅”，还要学会遵守偏好、解决复杂推理、控制输出成本和安全风险。

## 需要掌握的主题

### 1. SFT

SFT 让模型学会按照人类指令回答。它通常需要高质量 instruction-response 数据。

### 2. Preference Optimization

常见方法：

- RLHF：训练奖励模型，再用强化学习优化策略模型。
- DPO：直接用 chosen/rejected 偏好对优化模型。
- ORPO/KTO/IPO 等：不同偏好目标函数的变体。

你要会讲：

“DPO 的优势是绕过显式奖励模型，直接用偏好对优化语言模型；但数据质量、偏好分布和过优化仍然会影响效果。”

### 3. Reasoning Models

DeepSeek-R1、OpenAI o 系列这类 reasoning model 说明：推理能力可以通过 inference-time scaling 和 learning-to-reason 共同提升。

重点问题：

- CoT 越长是否一定越好？
- 推理过程是否 faithful？
- RL 会不会带来 reward hacking？
- 如何评测数学、代码、规划任务？
- 如何降低 reasoning token 成本？

## 可做研究方向

- 小模型的 reasoning distillation。
- CoT length control。
- Preference data quality。
- Reward hacking detection。
- Tool-use reasoning evaluation。
- 多语言 reasoning 数据构造。

## 项目说明

`project/tiny_dpo_loss.py` 不训练真实模型，而是用数字演示 DPO loss 如何鼓励 chosen response 相对 rejected response 概率更高。

运行：

```bash
cd 09_alignment_reasoning_research/project
python3 tiny_dpo_loss.py
```
