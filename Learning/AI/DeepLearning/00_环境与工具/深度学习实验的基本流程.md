# 深度学习实验的基本流程

## 一条可复用的训练链路

`问题定义 → 数据切分 → DataLoader → 模型 → loss → optimizer → 训练 → 验证 → 测试 → 记录与复现`。测试集只在方案冻结后使用；在验证集上反复调参，本质上会对验证集过拟合。

## 训练循环

```python
for epoch in range(num_epochs):
    model.train()
    for X, y in train_loader:
        X, y = X.to(device), y.to(device)
        optimizer.zero_grad()
        loss = criterion(model(X), y)
        loss.backward()
        optimizer.step()
    model.eval()
    with torch.no_grad():
        validate(model, val_loader)
```

`train()` 与 `eval()` 会改变 Dropout、BatchNorm 的行为；`torch.no_grad()` 不构建反向传播图，可节省验证阶段的内存。每个 batch 只能在一次 `backward()` 后再更新参数，否则梯度含义会混乱。

## 指标与基线

回归至少记录 MAE/RMSE；分类至少记录 accuracy，并在类别不平衡时记录 precision、recall、F1 或 PR-AUC。先构造朴素基线（预测均值、最常见类别、线性模型）；模型优于随机不等于模型有业务价值。

## 实验记录

每次实验只改变一个主变量，并记录：代码版本、数据版本、随机种子、模型结构、学习率/批量大小/轮数、训练与验证曲线、耗时、结论。最有价值的记录是“为什么本次失败”。

## 常见误区

- 把数据增强泄漏到验证/测试集；验证集通常只做确定性预处理。
- 只看最后一轮指标，而不看曲线是否震荡或过拟合。
- 不保存 checkpoint；至少保存验证指标最好的模型和配置。
