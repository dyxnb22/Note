# 循环神经网络

本章沿序列建模、语言模型、时间反向传播和实现边界理解基础 RNN，重点关注状态、梯度与训练稳定性。

## 序列模型、文本预处理与语言模型
序列数据有顺序：下一项依赖历史。语言模型估计 `p(x_t | x_<t)`，可按概率逐 token 生成文本。文本预处理包括规范化、分词/子词切分、词表、编码、批量化与掩码。

固定词表会产生未知词；子词方法在词表大小与泛化间折中，也是现代 tokenization 的前身。训练语言模型时，输入通常是前 t 个 token，标签是右移一位后的 token；评估常用 perplexity，越低代表平均预测概率越高。

最小练习：将一段文本构造成 `(X, Y)` 的下一个字符预测样本，验证 Y 相对 X 右移一位。

## RNN 与时间反向传播
RNN 以隐藏状态汇总历史：`H_t = f(X_t W_xh + H_(t-1) W_hh + b)`。它通过参数共享处理可变长度序列，但长距离依赖会使梯度沿时间反复相乘，导致消失/爆炸。

时间反向传播（BPTT）将计算图沿时间展开。截断 BPTT 只反传有限步，降低成本但损失部分远程依赖。训练生成模型常使用 teacher forcing：训练时输入真实前一个 token，推理时只能输入自己的预测，因此可能产生暴露偏差。

最小练习：对字符级 RNN 加梯度裁剪，比较不裁剪时是否出现 loss/梯度爆炸。

## RNN 从零实现与框架实现
从零实现时，显式创建输入到隐藏、隐藏到隐藏、隐藏到输出的参数，手动维护 state；框架实现使用 `nn.RNN`。两者要共享相同的数据、loss 和训练策略，才可以公平比较。

序列 batch 的 state 必须按 batch 维对齐；如果 batch 之间连续读取文本，可以传递 detach 后的 state；如果每个 batch 独立，必须重置 state。`detach()` 防止反向图无限延长。

最小练习：用两种实现训练字符语言模型，并实现 temperature 采样生成一段文本；解释温度提高后为何更随机。
