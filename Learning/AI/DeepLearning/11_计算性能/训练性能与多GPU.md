# 训练性能与多GPU

本章关注训练吞吐、显存、数值精度和多设备扩展。性能结论必须在相同模型、批量和硬件条件下测量。

## GPU、异步计算与混合精度
GPU 擅长大规模并行张量运算，但 CPU 向 GPU 传输、频繁小算子和同步读取会成为瓶颈。CUDA 操作通常异步提交；测时前需要同步，否则只测到提交开销。

混合精度以 FP16/BF16 执行多数矩阵运算，必要处保留 FP32。FP16 范围较小，常需 loss scaling；BF16 范围更接近 FP32，现代硬件上更稳。使用 `torch.autocast` 和 GradScaler，先验证数值一致性再追求速度。

## 多 GPU 训练与性能分析
数据并行让每张 GPU 持有模型副本、处理不同 batch，再同步梯度；PyTorch 优先使用 DistributedDataParallel。有效 batch size 变大后通常需重新调学习率和 warmup，不能假设单卡配置直接可用。

性能分析先区分数据加载、前向、反向、优化器、通信和显存限制。吞吐量以 samples/s 或 tokens/s 衡量，不能只看单 step 时间。多卡加速低于线性是常态，原因包括通信、负载不均和小 batch。

## 学习与验证

- 运行对应最小代码，记录数据、随机种子、模型、优化器和指标。
- 每次只改变一个变量，对比训练曲线、验证指标、吞吐和显存。
- 把失败归因到数据、目标函数、优化、容量或系统资源，而不是只换模型。
- 能用自己的话解释本章各机制之间的因果关系，再进入下一章。
