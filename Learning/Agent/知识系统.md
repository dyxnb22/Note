# 知识系统

知识系统负责让资料可靠地进入、更新、授权、检索、引用和退出系统。RAG 是它的一个消费方式；完整的知识系统还要解决来源、版本、权限、删除、质量和审计。

检索与生成算法见 [RAG](RAG.md)。

## 知识对象模型

建议区分：

```text
Source：外部来源及连接配置
Document：一个逻辑文档
DocumentVersion：某次不可变内容快照
Chunk：可检索和引用的片段
IndexRecord：某个索引/模型下的派生记录
Citation：回答对版本和位置的引用
```

不要把向量库中的 Chunk 当作唯一事实源。原文、版本、权限和删除状态应保存在可治理的系统中，索引是可以重建的派生物。

## Ingestion Pipeline

```text
发现变化
→ 拉取
→ 病毒/类型/大小检查
→ 解析
→ 标准化
→ 质量校验
→ 版本落库
→ Chunk 与 Metadata
→ 建立多种索引
→ 发布可检索版本
```

每一步应幂等，失败可重试并保留状态。新索引完全就绪前，不要让查询同时看到半更新的数据；可通过版本别名或原子切换发布。

## 来源与证据等级

来源要记录：

- 原始 URI、所有者和采集时间。
- 作者、发布时间和有效期。
- 权威等级和适用范围。
- 内容 Hash、解析器版本和处理日志。
- 是否允许用于回答、训练或外发。

官方规范、内部批准文档、个人笔记和自动生成摘要不能拥有相同权重。检索排序和回答展示应体现来源等级。

## 版本与新鲜度

DocumentVersion 应不可变。更新时创建新版本，索引记录绑定确切版本。查询可以默认选择当前生效版本，但引用必须保留历史版本标识，保证过去回答可复现。

需要定义：

- 多久同步一次。
- 如何发现删除、重命名和权限变化。
- 新旧版本冲突时哪个生效。
- 过期内容是降权、隐藏还是明确标记。
- 索引落后于原文时是否允许回答。

## 权限感知检索

授权决策至少依赖主体、租户、资源、动作和上下文。权限标签应随文档和 Chunk 一起进入检索过滤条件。

安全顺序：

```text
验证身份
→ 计算允许范围
→ 在检索层过滤
→ Rerank
→ 组装上下文
→ 生成
→ 引用再次校验
```

仅在生成后隐藏引用是不安全的，因为模型已经看到了无权内容。缓存键也必须包含权限和租户上下文，防止跨用户命中。

## Citation 对象

完整引用至少包含：

- `document_id`、`version_id`、`chunk_id`。
- 标题、来源 URI 和定位信息。
- 引用片段或支持范围。
- 检索和回答时间。
- 权限校验结果。

展示 URL 可能变化，因此内部稳定 ID 和版本不可省略。引用点击时还要重新鉴权，不能把短期可见性永久固化到公开链接。

## 删除与撤回

删除不是只从向量库删一条记录。需要处理：

- 原文和所有版本的保留政策。
- 向量、词法、缓存和备份中的派生数据。
- 已生成回答和审计记录。
- 法律保留与用户删除请求的冲突。
- 下游索引失败后的重试和证明。

可以通过 Tombstone 记录删除事实，并让查询立刻过滤；后台再清理所有派生副本。必须能审计删除传播是否完成。

## 数据质量

摄取阶段检查：

- 空文档、乱码和 OCR 置信度。
- 重复或近重复内容。
- 标题层级、表格和代码是否保留。
- 页面和段落定位能否回到原文。
- 敏感信息和恶意指令。
- 文档语言和领域分类。

低质量内容不应无条件进入主索引。可以隔离、降权或等待人工复核。

## 冲突知识

多来源对同一问题给出不同答案时，不要简单用相似度最高者覆盖。应结合权威性、生效时间、适用区域和版本，并在无法自动裁决时向用户展示冲突。

业务规则类知识可以增加“生效区间”和“替代版本”，避免旧政策仍被召回。

## 观测与运营

关键指标包括：

- 来源同步成功率和延迟。
- 解析失败、空文档和重复率。
- 当前版本与索引版本差距。
- 零结果率、低置信召回率和过期命中率。
- 权限过滤前后候选数。
- 引用点击、纠错和人工反馈。
- 删除传播完成时间。

每次回答应能反查使用了哪些版本、检索参数和权限范围。

## 灾难恢复

索引应可由版本化原文和处理配置重建。需要备份：

- 原始文档或受控引用。
- 文档与版本元数据。
- 权限映射。
- Chunk/解析器配置和模型版本。
- 发布状态和审计日志。

仅备份向量库不足以恢复来源、引用和权限语义。

## 验收清单

- 任一 Chunk 能否回到确切原文版本和位置？
- 权限变化多久能在检索结果中生效？
- 文档删除是否覆盖索引、缓存和备份策略？
- 索引能否从事实源重建？
- 冲突和过期内容是否有明确处理规则？
- 一次历史回答能否复现当时使用的知识版本？
