# Agent 版本与来源

> 最后核对：2026-07-22。这里集中管理容易变化的外部事实；正文优先讲稳定机制。

## 使用规则

- 模型 ID、价格、上下文窗口、区域可用性和弃用日期，运行前必须在 Provider 官方文档重新确认。
- 示例代码通过 `PRIMARY_MODEL`、`FAST_MODEL` 等配置项选择模型；文档中的具体 ID 只表示示例当时的环境，不构成推荐。
- SDK、框架和协议示例应锁定依赖版本，并在升级时运行固定回归集。
- 价格比较保存“输入量 × 输入单价 + 输出量 × 输出单价”的计算过程，不在概念文档复制易过期价目表。
- 第三方博客只用于发现线索；协议语义、API 参数和安全边界回到官方规范或源码验证。

## 当前 OpenAI 基线（2026-07-22）

- 新建的推理、工具调用和多轮工作流优先使用 Responses API；Chat Completions 示例只用于理解仍在维护的旧接口或兼容既有系统。
- 当前旗舰系列为 GPT-5.6：`gpt-5.6` 是指向 `gpt-5.6-sol` 的别名；`gpt-5.6-terra` 偏向成本与能力平衡，`gpt-5.6-luna` 偏向高吞吐和低成本。
- 教程正文不把具体模型 ID 写成业务默认值。运行示例时通过 `OPENAI_MODEL` 等环境变量显式指定，并在项目配置或实验记录中保存实际 ID。
- 以上信息核对自 [OpenAI 最新模型指南](https://developers.openai.com/api/docs/guides/latest-model)；能力、价格和可用区域仍需在运行前复核。

## 升级检查单

1. 记录旧版本、新版本、变更原因与回滚方式。
2. 检查 API schema、默认值、错误类型、流式事件和工具调用格式。
3. 在离线数据集比较任务成功率、安全拒绝、延迟和成本。
4. 检查缓存键、token 计算、结构化输出与 stop reason 等边界。
5. 小流量灰度，观察失败分布而不只看平均值。
6. 通过门禁后更新锁文件、示例配置及本页的核对日期。

## 文档维护约定

- 稳定原理写在专题正文；版本事实只在本页或项目配置中维护。
- 文档若必须保留历史价格或模型 ID，需同时标注日期和用途（复现实验，而非当前推荐）。
- 发现官方链接失效时，先更新本页，再检查引用该事实的专题文档。
