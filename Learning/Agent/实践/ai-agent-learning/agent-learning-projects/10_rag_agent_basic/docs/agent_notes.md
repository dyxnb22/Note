# Agent 学习笔记

Agent 是能围绕目标进行多步操作的 LLM 应用。它通常具备工具调用、状态管理和一定的循环控制能力。

Tool Calling 的关键点是：模型不直接执行函数，模型只决定要调用哪个工具以及参数。真正执行工具的是宿主程序。

LangGraph 使用图结构组织 Agent 流程。State 保存共享状态，Node 执行单步逻辑，Edge 决定下一步去哪里。

RAG 是 Retrieval Augmented Generation，也就是检索增强生成。它先从文档中检索相关片段，再把片段作为上下文交给模型回答。

不能把所有文档都塞给模型，因为上下文窗口有限，成本也会升高，而且无关内容会干扰回答。
