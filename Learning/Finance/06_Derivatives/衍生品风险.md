# 衍生品风险

## 1. 主题定位

这篇系统总结衍生品投资中的主要风险类型：市场风险、信用风险、流动性风险、操作风险和模型风险，以及历史上衍生品导致的重大损失案例。

学它之前最好先懂：期货基础、期权基础、杠杆与保证金、对冲与投机。

学完后可以继续学：风险预算（如何在组合层面管理这些风险）、高收益陷阱（14 模块）。

---

## 2. 核心概念

衍生品的风险来源多于普通资产，主要因为：
1. 高杠杆放大了市场波动的影响
2. 非线性收益使风险难以直觉量化
3. OTC 衍生品有交易对手违约风险
4. 复杂性使错误更难被及时发现

## 3. 核心机制

**风险类型一：市场风险（Market Risk）**

衍生品持仓因标的资产价格、利率、汇率等市场变量变化而产生的损失。

关键特征：
- 期权的非线性 Delta：标的价格不同幅度的变化，期权价格变化幅度不同（Gamma 效应）
- 杠杆放大：同样的标的价格变化，对衍生品持仓的影响可以是现货的数倍乃至数十倍
- Vega 风险：隐含波动率变化对期权价值的影响（持有期权的人面临 IV 下跌风险）

**风险类型二：信用（交易对手）风险（Counterparty Risk）**

OTC 衍生品（远期、互换、场外期权）的对手方可能无力履约。

关键特征：
- 交易所衍生品（期货、标准化期权）通过清算所净额结算，信用风险几乎消除
- OTC 衍生品的信用风险实质，取决于对手方信用评级和净敞口

2008 年金融危机中，AIG 因出售大量信用违约互换（CDS）而无力履约，成为系统性风险的传染节点。

**风险类型三：流动性风险（Liquidity Risk）**

不能在合理价格、合理时间内平仓（卖出头寸）的风险。

关键特征：
- OTC 衍生品流动性差，特别是非标准化合约，可能无法找到对手方
- 市场压力时，即使标准化期货的买卖价差也会急剧扩大
- 强制平仓时，大量同向持仓同时寻找对手方，引发踩踏

**风险类型四：操作风险（Operational Risk）**

内部流程、系统、人员错误，或外部事件导致的损失。

金融史上最著名的操作风险案例：
- 巴林银行（1995 年）：交易员 Nick Leeson 在大阪期货市场累计亏损 13.8 亿美元，银行倒闭。核心是内控缺失——同一人同时负责交易和后台。
- 法国兴业银行（2008 年）：Jerome Kerviel 通过欺骗性操作累计持有高达 700 亿欧元的未授权头寸，亏损 49 亿欧元。

**风险类型五：模型风险（Model Risk）**

依赖错误的定价模型或风险模型，导致对风险的错误估计。

关键特征：
- Black-Scholes 期权定价模型假设正态分布，低估尾部事件
- VaR 模型假设历史相关性在危机中不变（实际上危机时相关性急剧上升）
- 2008 年 CDO 定价模型错误地假设不同地区房价违约相关性极低

## 4. 分析框架

**使用衍生品前的风险检查清单：**

1. 市场风险：最坏情景（如大幅逆向移动 2-3 倍标准差）下的损失是多少？
2. 爆仓风险：杠杆率是多少？触发追加保证金的价格是多少？
3. 对手方风险：OTC 还是交易所？对手方信用评级？
4. 流动性风险：能否在需要时及时平仓？平仓成本预计多少？
5. 模型风险：使用的定价/风险模型的主要假设是什么？在极端情景下成立吗？
6. 操作风险：审批流程是否健全？有没有独立的风险监控？

---

## 5. 关键现象与观察点

**尾部风险（Tail Risk）**
市场上存在低概率但影响巨大的极端事件（黑天鹅）。正态分布大幅低估这类事件的概率。依赖正态分布计算的 VaR，在危机中系统性低估实际损失。

**相关性在危机中的急剧上升**
正常市场中，不同资产相关性低，分散化有效。但危机时，"所有东西都同时下跌"——资产相关性急剧上升，分散化大幅失效。依赖历史相关性构建的衍生品组合，在危机时风险远超模型预测。

## 6. 实际例子

**例 1：长期资本管理公司（LTCM，1998 年）**
LTCM 是由诺贝尔经济学奖得主参与创建的对冲基金，使用高度杠杆（约 25-30 倍）的固定收益套利策略。1998 年俄罗斯金融危机和日元急升，触发大规模平仓，LTCM 陷入流动性危机，美联储不得不组织华尔街银行联合救援。

教训：无论模型多精密，极端市场情景下的流动性危机 + 高杠杆 = 系统性崩溃风险。

**例 2：2008 年 CDS 系统性风险**
信用违约互换（CDS）作为信用风险转移工具，本身是合理的金融工具。但大量卖家（如 AIG）以极低保证金承担了大量信用风险，且彼此之间高度连接。当次贷危机来临，大量 CDS 同时触发，超出卖家赔付能力，引发系统性危机。

## 7. 常见误区

**误区 1：衍生品本身是危险的，应该禁止**
衍生品是工具，问题在于滥用（过高杠杆、缺乏对手方信用管理、内控缺失）。合理使用衍生品（如企业套期保值、组合对冲）是正当且有价值的风险管理手段。

**误区 2：复杂结构产品更安全（因为"分散化"）**
2008 年的 CDO²（CDO 的 CDO）被设计为"充分分散化"，结果相关性假设完全错误，在危机中一文不值。复杂性本身增加了模型风险，不是减少了它。

**误区 3：VaR 是可靠的风险度量**
VaR 告诉你在正常市场中，95% 的时间损失不超过多少，但对剩余 5% 的极端损失没有任何约束。VaR 是最低损失估计，不是最大损失估计。

## 9. 最小可执行练习

**练习 1（风险识别）**
某公司财务部门通过 OTC 互换，将 5 年期固定利率债务换成浮动利率，对手方是某商业银行。识别这笔交易面临的：市场风险（哪种变化不利）、信用风险（什么情景下发生）、操作风险（什么可能出错）。

**练习 2（LTCM 案例分析）**
LTCM 的杠杆约 25 倍，策略假设相关资产的利差会收敛（均值回归）。1998 年危机中，利差不但没收敛，反而急剧扩大（飞行至安全资产）。从市场风险、流动性风险、模型风险三个角度分析 LTCM 的失败原因。

**练习 3（尾部风险量化）**
某策略历史年化收益 10%，年化波动率 5%，正态分布假设下的 99% VaR = 约 10 万元（100 万本金）。但历史上实际出现过一次单月损失 30%。讨论：正态假设下的 VaR 多大程度上低估了真实尾部风险？

---

## 10. 一页总结

**五大风险类型**：
1. 市场风险（非线性 + 杠杆放大）
2. 信用 / 交易对手风险（OTC 特有）
3. 流动性风险（压力时无法平仓或成本极高）
4. 操作风险（人为错误、内控失效）
5. 模型风险（错误假设导致错误定价和风险估计）

**历史教训**：巴林银行（操作风险）、LTCM（杠杆 + 流动性 + 模型）、2008 年 AIG（交易对手风险的系统传染）。

**核心原则**：高杠杆在极端情景下可以快速摧毁本金；复杂不等于安全；VaR 是最低损失估计，不是最大损失上限。

## 11. 术语表

| 术语 | 解释 |
|------|------|
| 市场风险 | 资产价格不利变动带来的损失风险 |
| 交易对手风险 | OTC 衍生品对手方无力履约的风险 |
| 流动性风险 | 无法在合理价格、合理时间内平仓的风险 |
| 操作风险 | 流程、系统、人员错误导致的损失 |
| 模型风险 | 依赖错误假设的定价或风险模型导致的损失 |
| VaR（在险价值）| 给定置信水平（如 95%）下的最大损失，不代表最大可能损失 |
| 尾部风险 | 低概率但高影响的极端事件带来的损失风险 |
| CDS（信用违约互换）| 转移信用风险的 OTC 衍生品，"信用保险" |
