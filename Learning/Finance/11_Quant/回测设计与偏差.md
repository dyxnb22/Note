# 回测设计与偏差

## 1. 回测回答什么

回测是在历史信息集上模拟策略决策与成交，用来排除明显无效的假设、理解收益来源和暴露失败条件。它不能直接证明未来收益；越多研究决策由同一段数据驱动，结果越接近“对历史的拟合”。

可信结论必须同时说明：当时可见的数据、可交易的资产、决策规则、成交假设、成本、样本划分，以及研究过程中试过多少变体。

## 2. 最小研究协议

在运行回测前先冻结一页协议：

| 项目 | 必须记录 |
|---|---|
| 假设 | 预期机制、适用市场与失效条件 |
| 数据 | 来源、覆盖期、发布时间、修订规则、退市样本 |
| 信号 | 计算公式、观察时间、可交易时间 |
| 组合 | 选样、权重、约束、再平衡频率 |
| 执行 | 订单时间、成交价格、滑点、费用与容量 |
| 验证 | 样本内、验证集、最终样本外及 walk-forward 方案 |
| 搜索范围 | 测试过的因子、参数、市场和筛选规则数量 |
| 门槛 | 事先定义的收益、风险、稳定性和可实施性要求 |

标准流程：

```text
提出可证伪假设
→ 构建 point-in-time 数据集
→ 冻结信号与执行规则
→ 样本内开发
→ 验证集选型
→ 最终样本外只评估一次
→ 成本、容量和压力测试
→ 纸面交易或小规模实盘
→ 持续监控与归因
```

如果看过最终样本外结果后继续调参，这段数据已经变成训练信息，必须另找未见数据验证。

## 3. 时间与数据边界

每条数据至少区分三个时间：

- `period_end`：经济事实所属期间。
- `published_at`：市场首次可获得时间。
- `effective_at`：经过采集、清洗后策略可使用时间。

信号只能使用 `effective_at <= decision_at` 的记录。财报期末不等于披露日，修订后的宏观数据也不等于当时首次公布值。

资产池同样需要 point-in-time：每个历史时点只包含当时已上市且可交易的标的，并保留退市、停牌、更名和指数成分变更。直接用今天的成分股回溯历史会产生幸存者偏差。

执行顺序也要可实现。例如用收盘后才能计算的信号按同一收盘价成交，通常隐含未来信息；应使用下一可交易时点，或明确建模真实的竞价与延迟。

## 4. 主要偏差与防御

| 偏差 | 典型来源 | 防御 |
|---|---|---|
| 前视偏差 | 提前使用财报、修订数据、未来指数成分或当期成交价 | point-in-time 数据；逐字段检查发布时间与延迟 |
| 幸存者偏差 | 数据库只有当前存续公司或基金 | 纳入退市与终止产品；按历史资产池构建样本 |
| 多重检验 | 测试大量因子和参数，只报告最好者 | 记录完整搜索空间；控制假发现；独立样本外验证 |
| 过拟合 | 参数多、规则复杂、局部最优尖锐 | 降低自由度；检查邻域稳定性；walk-forward |
| 选择偏差 | 事后选择市场、时间段、资产或过滤条件 | 事前定义范围；报告被排除样本与理由 |
| 成本偏差 | 忽略费用、买卖价差、冲击和借券约束 | 用订单级或分层成本模型；做容量压力测试 |
| 实现偏差 | 复权、时区、缺失值、公司行动处理错误 | 数据契约、单元测试、抽样重放和双实现核对 |
| 发布偏差 | 只保留成功研究，失败实验消失 | 研究日志和实验注册表保留全部尝试 |

这些偏差不会简单相加，也不一定永远只高估表现；关键是它们让估计失去可解释性，不能用一个统一“折扣系数”补救。

## 5. 样本划分与 walk-forward

时间序列不能随机打乱后做普通交叉验证，否则未来观察可能进入训练集。

一种可执行结构是：

1. 早期窗口用于研究和拟合。
2. 中间窗口用于少量模型选择。
3. 最后窗口作为冻结的最终样本外。
4. 用滚动或扩展窗口重复“只用过去预测未来”，检查不同市场状态。

窗口长度取决于信号频率、有效样本量、市场制度和预期持有期，不存在通用的“30% 样本外”或“必须覆盖若干年”规则。应报告各窗口包含的行情状态、有效交易数和置信区间。

跨市场验证只有在机制和交易约束可比时才有意义。一个信号在另一市场失效，可能否定普遍性，也可能反映数据定义、成本或制度差异；需要先做归因。

## 6. 成本、滑点与容量

净收益至少扣除：佣金和税费、买卖价差、市场冲击、延迟、借券或融资成本。成本应随成交量占比、流动性、波动和订单方向变化，而不是所有交易使用同一常数。

至少报告三组情景：

- 基准：基于可观察历史条件的合理估计。
- 保守：更差的价差、延迟与冲击。
- 容量压力：资金规模或参与率上升后的表现。

高换手策略先看毛收益是否足以覆盖成本不确定性，再讨论复杂优化。详细建模见 [滑点、交易成本与容量](滑点交易成本与容量.md)。

## 7. 结果如何解读

不要只报告年化收益和 Sharpe。至少同时给出：

- 样本内、验证和最终样本外的收益与风险。
- 最大回撤、尾部损失、换手、暴露和容量。
- 按年份、市场状态、行业与信号分桶的稳定性。
- 参数邻域和成本情景下的结果分布。
- 基准、消融实验与因子/执行归因。
- 搜索过的策略数量及未成功实验。

样本外与样本内指标的比值可以描述衰减，但不能用固定阈值自动判定有效。指标可能接近零、为负或估计不稳定；结论应结合置信区间、试验次数、经济机制和可实施性。

多重检验不能只靠最朴素的 Bonferroni 修正解决。候选之间往往相关，而且研究流程包含许多未显式记录的选择。可以结合 false discovery rate、Deflated Sharpe Ratio、bootstrap 和严格保留集，但任何校正都不能替代完整实验日志。

## 8. 审查清单

- [ ] 研究假设和门槛是否在看结果前写下？
- [ ] 每个字段是否有 point-in-time 的可用时间？
- [ ] 历史资产池是否包含退市和不可交易状态？
- [ ] 信号产生与成交之间是否留出真实延迟？
- [ ] 数据清洗、复权、时区和缺失值是否有测试？
- [ ] 是否记录全部参数、因子与样本选择次数？
- [ ] 最终样本外是否只使用一次？
- [ ] 成本是否包含价差、冲击、融资/借券与容量？
- [ ] 结果是否对参数邻域、时间窗口和市场状态稳定？
- [ ] 是否有简单基准和逐项消融？
- [ ] 上线后是否能复现实盘与回测之间的差异？

## 9. 最小练习

审查下面的描述：

> 使用今天仍上市的公司构建 2010–2025 年样本；以季度末财务数据在季度末收盘成交；测试 80 个因子后保留 Sharpe 最高的三个；每笔交易按固定低费率扣费。

应至少识别：幸存者偏差、财报可得时间导致的前视偏差、成交时点不可实现、多重检验，以及缺少随流动性和规模变化的成本模型。然后为每项写出数据字段、测试或流程上的具体修复。

## 10. 一页结论

- 回测是历史模拟和反证工具，不是未来收益证明。
- 时间正确性优先于模型复杂度：只使用当时真实可见、可交易的信息。
- 样本外会因反复查看而失效；完整记录研究搜索空间。
- 成本、容量和执行约束属于策略定义，不是最后补上的折扣。
- 稳健性来自多种证据共同支持，不来自单一经验阈值。
