# 滑点、交易成本与容量

## 1. 主题定位

这篇讲量化策略中容易被忽视的"摩擦"：滑点、交易成本的各个组成部分，以及策略容量的概念和估算方法。这些是将理论 Alpha 转化为实际净收益的关键约束。

学它之前最好先懂：量化投资是什么、回测原则、回测偏差（交易成本低估偏差）。

学完后可以继续学：信号组合执行归因（在实盘中量化成本损耗）。

---

## 2. 核心概念

**交易成本（Transaction Cost）**
每次交易发生的总成本，包括显性成本（佣金、印花税）和隐性成本（买卖价差、冲击成本）。

**滑点（Slippage）**
实际成交价格与期望成交价格的差距。来源：① 市场报价变动 ② 买卖价差 ③ 大单对价格的冲击。

**市场冲击成本（Market Impact）**
大额交易本身对价格的推动效应。买入越多，价格被推得越高；卖出越多，价格被压得越低。规模越大，冲击越严重。

**策略容量（Strategy Capacity）**
在可接受的 Alpha 衰减程度内，策略能够有效管理的最大资金规模。超过容量后，冲击成本侵蚀了大部分 Alpha。

---

## 3. 核心机制

**交易成本的完整构成**

```
总交易成本 = 显性成本 + 隐性成本

显性成本：
  - 佣金（Commissions）：券商收费，A 股约 0.02%-0.05% 单边
  - 税费（Taxes）：A 股印花税 0.1% 单边卖出；港股印花税 0.1% 双边

隐性成本：
  - 买卖价差（Bid-Ask Spread）：最佳买价与卖价的差，流动性差的股票更宽
  - 市场冲击（Market Impact）：大单推动价格的成本，与规模正相关
  - 时机成本（Timing Cost）：等待执行期间价格变动的成本
```

**冲击成本的规模效应**

冲击成本与交易规模的关系通常是非线性的：

```
冲击成本（bps）≈ k × √(交易量 / 日均成交量)

其中 k 为市场系数（A 股大盘股约 50-100 bps 级别）
```

举例：交易量占日均成交量 1% 时，冲击约 5-10 bps；占 10% 时，冲击约 15-30 bps；占 50% 时，冲击可达 60-100 bps。

**策略容量的估算框架**

```
步骤 1：确定策略的目标 Alpha（年化，扣成本前）
步骤 2：估算每次调仓的换手率
步骤 3：计算每单位资产每年的冲击成本（与规模 N 有关）
步骤 4：令 Alpha = 冲击成本，解出 N
N 即为理论容量上限
```

实践中通常在理论容量的 30%-50% 处 Alpha 已经显著衰减，因此实际容量 ≈ 理论容量的 1/3。

**不同类型策略的成本特征**

| 策略类型 | 换手率 | 单次冲击影响 | 容量 | 成本敏感度 |
|---------|--------|------------|------|-----------|
| 高频统计套利 | 极高 | 低（单次小）| 极低 | 极高 |
| 因子量化（月调仓）| 中 | 中 | 中 | 中 |
| Smart Beta | 低（季/年）| 低 | 高 | 低 |
| 主观选股 | 低 | 低 | 较高 | 低 |

## 4. 分析框架

**回测中合理的成本假设（A 股参考）**

| 股票类型 | 单次换手成本（双边）| 说明 |
|---------|-----------------|------|
| 沪深 300 成分股 | 0.3%-0.5% | 流动性好，冲击小 |
| 中证 500 成分股 | 0.5%-0.8% | 流动性中等 |
| 中小市值股票 | 0.8%-2%+ | 流动性差，冲击大 |
| 小市值（后 20%）| 2%-5%+ | 慎用，容量极小 |

> 此部分可能随监管、交易所规则、产品设计变化，需以最新规则为准。

**策略容量的定性判断**

- 月换手率 > 50%（年换手 > 600%）→ 高成本策略，容量通常 < 5 亿
- 月换手率 10%-50% → 中等成本，容量通常 1-50 亿
- 月换手率 < 10%（年换手 < 120%）→ 低成本策略，容量通常 > 50 亿

## 5. 关键指标或关键变量

**年化换手率**
= 每年全部持仓变动的总比例（双边）。月调仓策略，每月换手 30% → 年换手率约 360%。

换手率 × 单次成本 = 年化成本损耗。这是将 Gross Alpha 转化为 Net Alpha 的核心公式。

**净 Alpha（Net Alpha）**
= Gross Alpha - 年化交易成本

只有净 Alpha 才是投资者实际获得的超额收益。高 Gross Alpha + 高换手成本 = 可能的净 Alpha 为零甚至负。

**容量系数（Capacity Ratio）**
= 当前管理规模 / 策略理论容量

容量系数 < 0.3：成本影响小，Alpha 基本完整。
容量系数 > 0.7：成本侵蚀显著，需要考虑降低规模或换策略。

## 6. 实际例子

**例 1：成本对净 Alpha 的侵蚀**
某因子策略 Gross Alpha = 8%（年化），月换手率 30%（年换手 360%）。
假设成分股平均单次双边成本 0.6%：
年化成本 = 360% × 0.6% = 2.16%
净 Alpha = 8% - 2.16% = 5.84%

如果改为集中在小市值股票，单次成本 1.5%：
年化成本 = 360% × 1.5% = 5.4%
净 Alpha = 8% - 5.4% = 2.6%（仅剩 1/3）

**例 2：规模对容量的影响**
某量化私募 1 亿规模时，每月换 30%（约 3000 万），持股 100 只，平均每只买卖 30 万——对大多数 A 股中小盘股，冲击极小。

扩张到 100 亿时，同等换手每月 30 亿，每只买卖 3000 万——对中等市值股票（日均成交 5000 万），单次占日成交量 60%，冲击成本可达 80-150 bps，几乎吞噬全部 Alpha。

**例 3：换手率优化降成本**
同等因子信号，月调仓换手率 30% vs. 每季度换手 10%（降低换手但略微牺牲信号及时性）：
- 月调仓：年成本 = 360% × 0.5% = 1.8%
- 季调仓：年成本 = 40% × 0.5% = 0.2%

成本降低 1.6%，但信号衰减（因子效应衰退导致）约减少 Alpha 0.8%。净效果：季调仓净 Alpha 更高。

## 7. 常见误区

**误区 1：回测中使用收盘价成交**
实际上无法以当天收盘价下单（收盘价在收盘那一刻才确定）。应使用下一交易日开盘价，或加入滑点假设。使用收盘价会系统性低估交易成本。

**误区 2：忽视冲击成本，只算佣金**
对于资金规模超过 1 亿的策略，冲击成本往往远大于佣金。只算佣金的成本估算严重失真。

**误区 3：认为策略容量可以无限扩张**
所有 Alpha 来源都有容量限制。规模越大，Alpha 越低，这是不可回避的物理约束（流动性有限）。任何声称"策略可以无限扩张"的说法都应当怀疑。

**误区 4：小市值因子在大规模资金下仍然有效**
小市值因子的超额收益在学术研究中显著，但学术研究是等权、小仓位的模拟。实盘中，试图用数十亿资金操作小市值股票，冲击成本会完全消灭 Alpha，甚至变成负 Alpha。

## 9. 最小可执行练习

**练习 1（成本计算）**
某月调仓策略，年换手率 480%，持仓 150 只中证 500 成分股，管理规模 5 亿。估算：① 年化显性成本（佣金+税费）② 年化冲击成本（中证 500 股票日均成交约 8000 万，每次交易占日成交量估算）③ 如果 Gross Alpha 为 10%，Net Alpha 是多少？

**练习 2（容量计算）**
某量化策略 Gross Alpha = 12%，年换手率 300%，持股 200 只，均为中证 500 成分股（日均成交约 6000-8000 万）。估算该策略在什么规模下净 Alpha 降至 5%，在什么规模下净 Alpha 降至 0？

**练习 3（换手优化权衡）**
设计一个因子策略的换手率测试：假设因子信号强度随时间衰减（1 个月后剩 70%，3 个月后剩 40%，6 个月后剩 20%）。在单次双边成本 0.6% 的条件下，月、季、半年调仓各自的预期净 Alpha 是多少？最优调仓频率是什么？

## 10. 一页总结

**交易成本构成**：显性成本（佣金+税费）+ 隐性成本（买卖价差 + 冲击成本）。冲击成本随规模非线性上升，是大规模策略的核心约束。

**净 Alpha = Gross Alpha - 年化成本**：年化成本 = 年换手率 × 单次成本。

**策略容量**：超过容量后冲击成本侵蚀 Alpha，实际容量约为理论容量的 1/3。

**核心原则**：用保守成本假设回测；高换手率策略对容量极度敏感；小市值因子在大规模下不可行。

## 11. 术语表

| 术语 | 解释 |
|------|------|
| 滑点（Slippage）| 实际成交价与期望价格的差距 |
| 市场冲击（Market Impact）| 大额交易推动价格的隐性成本 |
| 买卖价差（Bid-Ask Spread）| 最优买价与卖价的差，流动性溢价 |
| 年化换手率 | 每年全部持仓变动的总比例（双边）|
| Gross Alpha | 交易成本前的超额收益 |
| Net Alpha | 扣除交易成本后的实际超额收益 |
| 策略容量 | 可接受 Alpha 衰减下的最大可管理规模 |
