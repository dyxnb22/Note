# 经典排序与图算法

这篇用于补充题型笔记之外的基础算法速查，重点记住适用条件、复杂度和取舍，不必死记完整代码。

## 复杂度速查

| 算法 | 平均时间 | 最坏时间 | 额外空间 | 稳定性 |
|---|---:|---:|---:|---|
| 二分查找 | O(log n) | O(log n) | O(1) | - |
| 插入/冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(log n) 平均，退化时 O(n) | 通常不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 |
| 桶/基数排序 | 取决于桶和位数 | 取决于数据分布 | 通常 O(n + k) | 可稳定实现 |

## 二分查找的前提和边界

二分查找要求数据已经按同一比较规则有序。每轮比较中点，根据目标与中点的关系保留一半区间；实现时先明确区间是 `[left, right]` 还是 `[left, right)`，并单独处理“第一个满足条件”和“最后一个满足条件”的边界。

常见变体：

- 查找精确值。
- 查找第一个 `>= target` 的位置，即 lower bound。
- 查找第一个 `> target` 的位置，即 upper bound。
- 在答案范围上二分，例如“最小可行容量”或“最大可行速度”。

## 常见排序怎么选？

- 数据量小或接近有序：插入排序简单，移动成本可控。
- 需要稳定排序：归并排序，或使用语言库提供的稳定实现。
- 内存受限且追求最坏复杂度：堆排序。
- 平均性能优先：快速排序，但要随机化基准或使用三数取中，避免退化到 O(n²)。
- 数据是有界整数：桶排序或基数排序可能绕开比较排序的 O(n log n) 下界，但要承担额外空间和分布假设。

快速排序的核心是“选基准并分区”，归并排序的核心是“分别排序后合并”；不要把两者都只记成 O(n log n)，它们的空间、稳定性和最坏情况不同。

## 最短路径算法如何选择？

| 场景 | 算法 | 关键条件 |
|---|---|---|
| 无权图 | BFS | 边权都相同，按层扩展 |
| 非负权单源最短路 | Dijkstra | 不能有负权边，常用优先队列 |
| 有负权边的单源最短路 | Bellman-Ford | 可检测负环，复杂度较高 |
| 全源最短路 | Floyd | 动态规划，适合点数较少的稠密图 |

Dijkstra 每次确定当前距离最小且未确定的节点；如果存在负权边，这个“确定后不再变小”的前提会失效。

## 最小生成树怎么做？

最小生成树针对连通无向带权图，目标是用 `V - 1` 条边连通所有顶点且总权重最小。

- **Kruskal**：按边权从小到大尝试加入，用并查集判断是否形成环；适合边较少的图。
- **Prim**：从一个顶点出发，每次选择连接已选集合和未选集合的最小边；适合边较多或需要从邻接表扩展的图。

两者都要保证最终图连通；不连通时得到的是最小生成森林。

## 回溯、剪枝和动态规划的关系

- 回溯：沿着“选择 → 递归 → 撤销选择”搜索解空间，适合排列、组合、棋盘和约束满足问题。
- 剪枝：在继续搜索前判断当前状态不可能产生有效解，尽早返回；排序候选、记录已访问状态和设置上下界都可以剪枝。
- 动态规划：当子问题重复且存在最优子结构时，用状态和转移保存中间结果，避免回溯重复计算。

最大子数组和可用 Kadane 算法：维护“以当前位置结尾的最大和”，在“延续旧区间”和“从当前元素重新开始”之间取最大值。最长公共子序列的二维状态与转移见 [[多维动态规划_done]]。
