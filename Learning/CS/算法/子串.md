# 子数组与子串

子数组和子串都要求元素在原序列中连续。先识别目标是“计数、最短/最长、极值还是匹配”，再选择前缀和、滑动窗口、单调队列或字符串算法。

## 方法选择

| 问题信号 | 常用方法 | 关键前提 |
|---|---|---|
| 区间和等于某值，允许负数 | 前缀和 + 哈希计数 | 统计此前出现过的 `prefix - target` |
| 最长/最短且约束可通过移动左端恢复 | 滑动窗口 | 必须证明窗口约束的单调性 |
| 每个定长窗口的最大/最小值 | 单调队列 | 队列保存仍可能成为最优值的下标 |
| 查找模式串或重复结构 | KMP、Z、哈希等 | 根据是否需精确匹配选择 |

## 前缀和计数：LeetCode 560

设当前位置前缀和为 `prefix`。若此前某个前缀为 `prefix - k`，两者之间的子数组和就是 `k`。哈希表保存“前缀和值 → 出现次数”，初始化 `{0: 1}` 才能计入从下标 0 开始的答案。

```text
count = {0: 1}
prefix = answer = 0
for x in nums:
    prefix += x
    answer += count[prefix - k]
    count[prefix] += 1
```

时间 `O(n)`、空间 `O(n)`。因为负数会破坏窗口和的单调性，这题不能普遍用双指针。

## 单调队列：LeetCode 239

队列保存下标，且对应值从队头到队尾单调递减：

1. 新元素进入前，弹出队尾所有不大于它的元素。
2. 弹出已经离开窗口的队头下标。
3. 窗口形成后，队头就是最大值。

保存下标而不是值，才能正确判断元素是否过期，也能处理重复值。每个下标最多入队、出队一次，总复杂度 `O(n)`。

## 最小覆盖：LeetCode 76

维护目标字符频次、窗口频次，以及当前满足要求的字符种类数。右端扩张直到覆盖全部要求；随后不断移动左端并更新最短答案，直到窗口不再合法。

固定长度 `128` 只适合题目明确限制为 ASCII 的情况。通用字符集应使用哈希表；不要把 ASCII 128、扩展编码和 Unicode 混为一谈。

## 常见错误

- 把子序列误当作连续区间。
- 前缀和哈希忘记初始化零前缀。
- 单调队列存值，导致重复元素和过期判断错误。
- 最小覆盖只比较字符种类，不比较每个字符所需次数。
- 未证明单调性就套滑动窗口。
