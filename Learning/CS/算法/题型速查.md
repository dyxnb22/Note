# 算法题型速查

这里集中保留旧 `_done` 草稿中的可迁移模式；具体题目仍按题型文件维护。做题时先识别不变量，再决定数据结构和复杂度。

## 常见模式

| 模式 | 识别信号 | 核心做法 | 代表题目 |
|---|---|---|---|
| Hash / 计数 | 查补数、分组、频次或前缀状态 | `Map` 存补数/计数；`Set` 判断存在性 | 两数之和、字母异位词、最长连续序列 |
| 双指针 | 数组有序、原地覆盖、两端收缩 | 快慢指针或左右指针；明确移动条件 | 移动零、三数之和、盛水容器 |
| 滑动窗口 | 连续区间满足/违反条件 | 右指针扩张，左指针恢复不变量；频次用计数表 | 无重复子串、最小覆盖子串、字母异位词 |
| 前缀和 / 单调队列 | 区间查询、连续和、窗口最值 | 前缀和降为 O(1) 查询；单调队列维护候选最优值 | 和为 K、滑动窗口最大值、区间和 |
| 数组 / 区间 | 连续子数组、区间合并、原地变换 | Kadane；按左端排序合并；利用翻转/置换 | 最大子数组和、合并区间、轮转数组 |
| 矩阵 | 边界、方向、行列有序 | 四边界收缩；分层遍历；从右上角排除行/列 | 螺旋矩阵、旋转图像、搜索二维矩阵 |
| 贪心 | 局部选择能证明不伤害未来 | 先写覆盖范围、最早结束或最小代价，再证明可达性 | 跳跃游戏、划分字母区间、加油站 |
| 回溯 | 组合、排列、路径、约束满足 | 选择 → 递归 → 撤销；按树层去重；剪枝 | 全排列、组合总和、单词搜索、N 皇后 |
| 动态规划 | 子问题重叠且有明确状态 | 先定义 `dp` 含义，再写转移和初始化 | 打家劫舍、零钱兑换、LCS、股票状态 |

## DP 方向判断

- 0-1 背包：每个物品一次，容量从后往前；完全背包：可重复使用，容量从前往后。
- 序列问题先区分“连续”和“子序列”；前者失配常归零，后者在跳过元素和匹配之间取最优。
- 股票、括号和树形问题先把“持有/卖出”“当前位置结尾”“偷/不偷”等状态写出来，再压缩空间。

## 复习检查

1. 目标是查找、计数、最值、可行性还是枚举全部方案？
2. 数据是否有序？结果是否要求连续、原地或保持相对顺序？
3. 循环不变量是什么，指针/状态何时推进，何时回退？
4. 重复、负数、空输入、边界和溢出如何处理？
5. 能否说清时间复杂度、空间复杂度及为什么不能更简单？
