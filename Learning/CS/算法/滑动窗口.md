# 滑动窗口

滑动窗口用于处理连续区间：右端扩张获取新元素，左端收缩恢复约束，并在移动过程中维护区间状态。它通常把枚举所有区间的 `O(n²)` 降为 `O(n)`，前提是左右指针都只单向移动。

## 两种基本类型

### 固定长度窗口

窗口大小为 `k`。每次加入右侧元素、移除离开窗口的左侧元素，再读取当前状态。

```text
先建立前 k 个元素的状态
记录答案
for right = k .. n-1:
    加入 a[right]
    移除 a[right-k]
    更新答案
```

适合定长子数组和异位词匹配。LeetCode 438 可维护两个字符计数数组；窗口达到 `len(p)` 时比较频次，而不是枚举排列。

### 可变长度窗口

右端持续扩张；当窗口违反约束时，循环移动左端直到重新合法。

```text
left = 0
for right in range(n):
    加入 a[right]
    while 窗口不合法:
        移除 a[left]
        left += 1
    用当前窗口更新答案
```

更新答案的位置取决于题意：求“最长合法窗口”通常在恢复合法后更新；求“满足条件的最短窗口”通常在条件成立时一边收缩一边更新。

## 为什么能做到 O(n)

不要被嵌套的 `for + while` 误导。右指针最多移动 `n` 次，左指针也最多移动 `n` 次，因此总移动次数是 `O(n)`。窗口状态必须支持增量加入和移除；若每步重新扫描整个窗口，复杂度会退化。

## 适用前提

窗口方法需要一种可维护的单调约束：右扩后，移动左端能让窗口朝“重新合法”的方向变化。例如全为正数时，区间和随右扩不减，LeetCode 209 可安全收缩。

如果数组允许负数，“和至少为 target”不再满足这个单调性，普通滑动窗口会漏解；应考虑前缀和加单调队列等方法。先证明窗口为何可以丢弃左端，再写模板。

## 三个典型状态

- LeetCode 209：维护窗口和；条件成立时收缩并更新最短长度。仅适用于题设中的正整数。
- LeetCode 3：维护字符最近位置或频次；出现重复时推进左端，求最长无重复子串。
- LeetCode 438：维护定长窗口的字符频次与目标频次，窗口每移动一步做一次增删。

## 常见错误

- 把 `if` 写成 `while` 的反面：一次收缩不足以恢复约束。
- 先移动指针再移除元素，导致删错位置。
- 混淆窗口 `[left, right]` 与 `[left, right)`，产生长度差一。
- 没处理空数组、`k > n`、目标为空等边界。
- 未证明单调性，机械套模板到包含负数或非连续区间的问题。

## 解题检查单

1. 题目是否要求连续区间？
2. 窗口是固定还是可变长度？
3. 状态如何在加入/移除一个元素时更新？
4. 何时收缩，何时记录答案？
5. 左端移动后，被丢弃的区间为何不可能产生更优解？

最后用最小数组、全相同元素、窗口覆盖全数组和无解情况手动走一遍指针。
