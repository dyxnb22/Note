**LeetCode 45. 跳跃游戏 II 最小跳跃次数**。`curDistance` 是当前这一步能走到的边界；`maxDistance` 是在这个边界内继续选点后，下一步能走到的最远位置。走到边界时，就必须跳一次，并把边界扩展到 `maxDistance`

**LeetCode 763. 划分字母区间** 划分字符串，同一字母最多出现在一个片段中。先记录每个字符**最后出现的位置**，然后遍历字符串，不断更新当前片段必须到达的最远边界；当当前位置等于这个边界时，说明这个片段可以切开

**LeetCode 376. 摆动序列** 相邻元素的差值如果正负交替出现，就叫摆动序列，求 摆动序列 的 最长子序列的长度 （不要求连续）。`prediff` 记录上一次非零有效趋势，`curdiff` 记录当前趋势；当 `curdiff` 非零，并且与 `prediff` 方向相反，或者 `prediff` 还处于初始状态 `0` 时，说明可以加入一个新的峰或谷，答案加一

`curdiff` 不能为 0；`prediff` 可以为 0，是为了处理“还没有有效趋势”的初始状态。`if (curdiff > 0 && prediff <= 0 || curdiff < 0 && prediff >= 0)`     
`int prediff = 0; int curdiff = 0; curdiff = nums[i+1] - nums[i];`

**LeetCode 1005. K 次取反后最大化的数组和** 先尽量把负数变正数；如果还剩奇数次取反，就把当前绝对值最小的数取反(先把数组和求出来，然后对翻转后的数组重排序，`sum - 2 * nums[0]`)

**LeetCode 134. 加油站**（第 i 个加油站有汽油 gas[i] 升，从第 i 个加油站开往第 i+1 个加油站需要消耗汽油 cost[i] 升。从其中的一个加油站出发，开始时油箱为空，是否存在起点，可以按顺序绕环路行驶一周）
curSum 表示从当前候选起点出发，到当前位置的剩余油量。如果 curSum < 0，说明当前起点失败，并且它到当前位置之间的所有点都不能作为起点（到达之前的任意一个点，油量一定是非负的，重新从这个点出发，等于之前的油量去掉，cursum反而减少了，所以不能作为起点）。所以起点直接更新为 i + 1。totalSum 表示全局总油量是否够绕一圈。如果 totalSum < 0，返回 -1；否则返回贪心找到的 index。只要 totalSum >= 0，就必定存在一个合法的起点(整圈总油量不少于整圈总消耗)

**LeetCode 135. 分发糖果**（ratings表示每个孩子的评分，相邻两个孩子中，评分更高的那个会获得更多的糖果；每个孩子至少一个）从左往右看，当前孩子评分比左边高，那么当前孩子糖果数必须比左边多 1；从右往左看，如果当前孩子评分比右边高，那么当前孩子糖果数必须比右边多 1（`result[i] = Math.max(result[i + 1] + 1, result[i]) `不破坏第一遍已经满足的左边关系）

**LeetCode 406. 根据身高重建队列** 数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。按照这个规则重新排序
身高从大到小排（身高相同k小的站前面），然后构造一个链表存储结果，que.add(p[1],p) 原来 index 位置以及它后面的元素全部往后移动。插入时，如果同位置已有元素，它会往后移；由于当前人是按身高从高到低处理的，当前人不会比已有元素高，所以不会影响已有元素的 `k` 结果。当前人插入到 index = k 的位置，也保证了他前面正好有 k 个身高大于等于他的人。

**LeetCode 56. 合并区间 **合并所有重叠的区间。仅按每个区间的起始点排序即可。用 curStart 和 curEnd 维护当前正在合并的区间；遇到重叠就扩展右边界，遇到不重叠就把当前区间加入结果（`es.add(new int[]{curStart, curEnd});`），并开启一个新区间。最后别忘了加入最后一个维护中的区间。
